# backend/docs/mustpass

코드를 작성하기 **전에**, 그 코드가 반드시 만족해야 하는 **불변식**을 사람·AI가 읽기 좋게 적어 두는 곳. (CLAUDE.md 규칙 2·3)

> **데모 제품이므로 테스트 코드로 강제하지 않는다.** mustpass 는 "믿고 가는 문서"다.
> 구현 전에 여기 적힌 항목을 읽고, 그에 맞춰 코드를 작성한다. (검증 테스트 없음 → [.claude/rules/testing.md](../../.claude/rules/testing.md))

## 포맷

도메인별로 **`NN-도메인.md`** 파일 1개. (프론트 [`frontend/docs/mustpass`](../../../frontend/docs/mustpass) 와 동일하게 마크다운으로 — 표·링크가 살아 읽기 좋다.)

각 파일 권장 구조:

```markdown
# NN. {도메인} — mustpass

> 목적: 이 도메인 코드가 반드시 지켜야 할 불변식을 개발자가 바로 넣을 수 있게 확정.
> 근거: (PRD 04 §n) = SSOT, (#4) = 백엔드 이슈.

## 불변식 (must)
- 연결할 사람(personId) 없이는 기록이 저장되지 않는다 — 400.
- 기록 저장 시 대상 인물의 마지막 만남·기록 수가 갱신된다.
- "1년 전 오늘"은 같은 월·일 기록이 없으면 빈 결과를 반환한다.

## 계약·엣지 (표로 상세)
| 상황 | 입력 | 기대 결과 |
|---|---|---|
| personId 누락 | POST /api/v1/events | 400 + 메시지 |
| ...  | ... | ... |
```

## 규칙 (CLAUDE.md와 정합)

- **PRD 프로즈를 복사하지 않는다.** 코드로 표현 못 하는 **검증 가능한 불변식**만 짧게.
- 구현 전에 읽고, 구현이 이 항목들을 만족하도록 작성한다.
- 동작(계약)이 바뀌면 여기부터 고친다. (문서가 코드에 가장 가까운 계약)

## 1. 제품 개요

### 1.1 한 줄 정의
나를 중심으로 한 인간관계를, 고정된 연락처가 아니라 **시간에 따라 변하는 흐름(연대기)** 으로 기록·시각화해 언제든 다시 꺼내보게 하는 **사적인 관계 도감**.

### 1.2 배경 — 왜 필요한가
- 관계 기록이 카카오톡·캘린더·사진첩·SNS에 **흩어져** 한눈에 볼 수 없다.
- 사람을 저장해도 "어디서·어떤 사이였는지" **맥락과 서사는 남지 않는다.**
- 소원해진 사람은 **알아차릴 계기 자체가 없어** 자연스럽게 잊힌다.
- 시간이 지나면 "그때 친했던 사람"이라는 감각만 남고 **왜 가까웠는지 단서가 사라진다.**
- → 연락처엔 사람이, 카톡엔 대화가, 사진첩엔 순간이, 캘린더엔 약속이 있지만 **사람 중심의 관계 흐름에 답을 주는 서비스는 아직 없다.**

### 1.3 제품 목표 (What we're building)
1. 사람별로 **만남·연락·추억·기념일**을 한 화면에서 본다.
2. 관계를 **시간축 위의 이야기 곡선/피드**로 시각화한다.
3. **30초 안에** 부담 없이 그날을 기록하게 한다.
4. **"1년 전 오늘"** 같은 회고로 잊은 인연을 다시 꺼내준다.
5. 관계 기록을 통해 **"나는 어떤 사람인가"** 를 돌아보게 한다.

### 1.4 컨셉 & 차별점 — 연대기(Chronicle)
경쟁 서비스(Dex·Clay·YourPond)는 대부분 *관계 그래프 + 프로필 + 기록 폼*으로 수렴한다. 관계도감은 **시간축을 앱의 뼈대**로 삼아 차별화한다.
- **시간이 주인공** — 세로로 흐르는 시간 위를 사람이 흐른다.
- **멀어진 관계 = 공백(점선 구간)** 으로 시각화. ("연락하세요" 알림이 아니라 *빈 자리 자체*를 보여준다.)
- **인물 = 프로필이 아니라 관계 온도가 오르내린 이야기 곡선.**
- **기록 = 폼이 아니라 "끊긴 이야기를 잇는 행위"** ("연대기에 새기기").

> **비-목표(Non-goals):** 보여주기식 SNS가 아니다 · 소통/메신저 도구가 아니다 · 인맥 영업 CRM이 아니다. **오직 "나"와 그 사람 사이의 사적인 관계 자각·회고 도구.**

## 2. 타겟 & 페르소나

### 2.1 주 타겟
- 고마운/소중한 관계를 잘 챙기고 싶은 사람 (특히 **회사의 옅은 관계**까지).
- 바쁨 속에서도 소중한 사람과의 변화·추억을 남기고 싶은 **감성적 기록 습관**의 사용자.
- 친구·동료 관계가 쉽게 바뀌는 **20·30대 직장인/사회 초년생**.
- 관계를 쉽게 잊는 사람 / 관계·나 자신을 **시각화·회고**하고 싶은 사람.

### 2.2 대표 페르소나 — "지우" (앱 내 기본 샘플 사용자)
| 항목 | 내용 |
|---|---|
| 프로필 | 20·30대 직장인. 사회생활로 인간관계가 자주 바뀜 |
| 니즈 | 손 많이 안 대고도 소중한 관계를 챙기고, 잊은 인연을 다시 떠올리고 싶음 |
| 페인 | 기록은 귀찮고, 소원해진 관계는 알아챌 계기가 없음 |
| 원하는 결과 | "나는 오래된 관계를 오래 지키는 사람"임을 데이터로 확인 |

## 3. 성공 지표 (데모/MVP 기준)

> 이번 스코프는 해커톤 MVP·데모다. 지표는 정량 KPI보다 **"데모가 Wow를 전달하는가"** 로 판정한다.

- **핵심 데모 3컷이 매끄럽게 흐르는가**
  1. 홈에서 **소원해진 관계를 스스로 발견** (관계 지도 + "1년 전 오늘" 플로팅)
  2. 인물의 **연대기/타임라인에서 추억을 다시 봄** (온도 곡선 · 공백 · 이벤트 피드)
  3. **30초 기록**으로 그 공백을 잇고, **인사이트에서 "내가 어떤 사람인지" 확인**
- **콜드스타트 회피:** 첫 진입 시 **샘플 데이터(지우 + 민지·현우·서연 등)** 가 채워져 있어 빈 화면이 없다.
- **미구현 버튼은 「준비중입니다」** 로 명시 (무반응 = 버그처럼 보임).

## 4. 디자인 원칙 (공통 UI 규칙)

**방향: 연대기(시간 주인공) × 블랙 & 화이트 에디토리얼.** 유채색 0에 가깝게, 흑·백·회색 + 타이포·여백·둥근 형태로 "따뜻함·힐링"을 표현. SNS의 요란함과 반대되는 담백·사적인 도감.

| 요소 | 규칙 |
|---|---|
| 색 | 배경 `#ffffff` / 패널 `#fafafa` · 플레이스홀더 `#f2f2f2` / 테두리 `#e6e6e6` / 잉크(강조·버튼) `#1c1c1c` / 서브 `#8c8c8c` · 흐림 `#b5b5b5` |
| 악센트(최소) | 마커 하이라이트(연녹) · 모서리 파스텔 블롭 정도만 허용 |
| 아바타 | **모노그램(이니셜)**. 사진 자리는 회색 플레이스홀더 |
| 일러스트 | 얇은 라인(1.5px) 그림일기 삽화 + 사건별 아이콘 뱃지. 스틱맨·과한 채색 금지 |
| 시간축 | 세로 스파인(2px `#e4e4e4`), 노드 채움=기록됨 / 비움=공백·시작점. **공백은 점선 스파인** |
| 대비 | 선택·활성 = 검정 채움(흰 글씨) / 비활성 = 흰 배경·회색 테두리. 강조 단어는 밑줄 |

## 6. 화면별 요구사항

> 각 화면: **목적 → 구성 요소 → 상호작용 → 데이터 → 인수 조건(AC)**.
> 우선순위(P1/P2/P3)는 `docs/14` 기준. P1=데모 필수 뼈대, P2=있으면 강해짐(무거움), P3=여유 시.

### S0. 앱 셸 · 내비게이션 `[P1]`
- **구성:** 하단 탭 5개 + 중앙 검정 원형 `＋` FAB. 상단 상태바(시간·점3개는 목업 장식).
- **상호작용:** 탭 선택 시 검정 채움 강조. `＋` = 기록 작성(S4) 진입.
- **AC:** 모든 화면에서 동일 위치·동일 라벨의 탭이 보이고, 현재 화면이 활성 표시된다.

**구성 요소**

|---|---|---|

| 인물 노드 | **모노그램** 원. 친밀도에 따라 **실선(가까움)/점선(옅음)** 연결, 흐린 노드=옅은 관계 | P1 (F-21) |
| 즐겨찾기 강조 | ★ 표시 인물은 **이중 테두리**로 강조 | P1 (F-62) |

**데이터:** 인물 목록(이름·이니셜·즐겨찾기·친밀도 등급), 마지막 상호작용일(공백 판정용), 1년 전 오늘 매칭 기록.

**AC**
- [ ] 나 노드 중심, 최소 5인 이상 샘플 인물이 친밀도별 실선/점선으로 그려진다.

**구성 요소**
| 요소 | 설명 | 우선순위 |

### S3. 사람별 타임라인 (연대기)
> **목적:** 한 사람과의 관계를 **시간축 이벤트 피드**로 되짚는다. 연대기 컨셉의 핵심 화면.

**구성 요소**
| 요소 | 설명 | 우선순위 |
|---|---|---|
| 헤더 | 뒤로(‹) · "{이름}와의 이야기" · "{연도}년부터 · N번 만남" | P1 |
| 요약 배너 | 모노그램 + **함께한 기록 N개 · 마지막 만남 N일 전** (마커 하이라이트 배경) | P1 |
| 활동 흐름 차트 | **만남 / 연락 / 추억** 3레인 × 월별 점. 점 탭 시 그날 기록만 필터 | P2 (F-66) |
| 이벤트 피드 | 세로 스파인 + 노드(아이콘 뱃지) + 카드 | P1 (F-30) |
| 이벤트 카드 | 제목 + **카테고리 태그**(만남/연락/기념일/프로젝트/기타) + **언제 / 왜 만났는지 / 무엇을 했는지** + 미니 일러스트 | P1 (F-68, F-67) |
| 마커 하이라이트 메모 | 특별 메모(예: 🔒 축의 10만원) 연녹 pill | P3 (F-72) |
| FAB | 우하단 ＋ = 이 사람과의 기록 작성(S4) | P1 |

**상호작용**
- 활동 흐름 차트의 점 탭 → 해당 시점 기록으로 스크롤/필터(P2).
- 이벤트 카드 탭 → 상세(있으면) / FAB → S4(대상 인물 프리셋).

**데이터:** 이벤트[] (일자·카테고리·제목·언제/왜/무엇·아이콘·마커메모·연결사진), 월별 만남/연락/추억 집계.

**AC**
- [ ] 이벤트가 최신→과거 순으로 세로 스파인에 노드+카드로 쌓인다.
- [ ] 각 카드에 카테고리 태그와 언제/왜/무엇 구조 필드가 보인다.
- [ ] 공백 구간은 점선 스파인으로 구분된다. *(공백 판정 로직 → O-2)*
- [ ] 활동 흐름 차트(P2 미구현 시) 자리엔 정적 이미지 또는 「준비중」.

### S4. 상황 기록 작성
> **목적:** 만남 직후 **30초 안에** 감정 칩·최소 입력으로 그날을 남긴다. (연대기에 "새기기")

**구성 요소**
| 요소 | 설명 | 우선순위 |
|---|---|---|
| 헤더 | 뒤로(‹) · "새 기록" · **저장** | P1 |
| 대상·리드 | 모노그램 + "오늘 {이름}랑 어땠어요?" + "감정만 골라도 돼요. 세 줄이면 충분해요." | P1 (F-05) |
| 감정 칩 | 반가움/뭉클/편안/즐거움/고마움/그냥 — **다중 선택**, 선택 시 검정 채움 | P1 (F-01) |
| 제목 + 카테고리 | 제목 입력(플레이스홀더) + 우측 카테고리 드롭다운(만남 등) | P1 (F-68) / 본문 3줄·장문 P3(F-02/03) |
| 날씨 칩 | 맑음/흐림/비/쌀쌀/더움 — 단일 선택 | P3 (F-69) |
| 함께한 사람 | {이름}/혼자였던 시간/다른 친구/가족 — 태그 | P1 (F-70) |
| 사진 첨부 | 최대 N장, 각 항목 삭제(×) + ＋ 추가 | P1 (F-06) |

**상호작용**
- 감정 칩 다중 토글 / 사람·날씨 칩 선택.
- "저장" → 이벤트로 저장되어 해당 인물 S3 타임라인·S2 최근 함께한 일에 반영.

**데이터:** 기록(대상인물·일자·감정[]·제목·카테고리·날씨·함께한사람[]·사진[]·본문(선택)).

**AC**
- [ ] 감정 칩만 골라도 저장된다(제목·본문 없이 가능).
- [ ] 저장 시 대상 인물의 타임라인/프로필에 즉시 반영된다.
- [ ] 사진은 추가·삭제할 수 있다(데모는 플레이스홀더 가능).
- [ ] 위 흐름을 **1분 내(체감 30초)** 로 완료할 수 있다.

> **결정 근거(중요):** 그림일기 컨셉(`docs/13`)의 "AI 그림 변환·원고지" 포맷은 **이번 스코프에서 제외**하고, 최종 확정 스케치대로 **칩 기반 폼**으로 만든다. 직접 그리기 강제는 "30초 부담 없는 기록" 기둥을 깨기 때문. (→ O-4)

### S5. 나 · 인사이트
> **목적:** 관계 기록을 통해 **"내가 어떤 사람인지"** 돌아본다. 나 자신과의 관계까지.

**구성 요소**
| 요소 | 설명 | 우선순위 |
|---|---|---|
| 프로필 헤더 | 모노그램 + 이름 + "관계도감 N년차 · 함께한 사람 N명" | P1 (F-75) |
| 관계 성향 히어로 | **검정 카드** — "당신은 {…한} 사람이에요." + 보조 한 문장(강조 단어 밑줄) | P1 (F-43) |
| 스탯 3칸 | 함께한 사람 · 새긴 기록 · 이번 달 챙김 | P1 |
| 성향 자세히(4축 바) | 오래된 관계 유지 / 새 관계 확장 / 먼저 연락 / 기록 꾸준함 — 막대 + 등급 | P2 (F-76) |
| 자주 느낀 감정 | 감정 칩 + 횟수 뱃지 (반가움38 등) | P2 (F-46) |
| 이번 주 감사일기 | 인용문 + "N월 M째 주 · {대상}에게" | P3 (F-49) |

**상호작용:** 대체로 조회 전용. (감사일기 작성 진입은 P3.)

**데이터:** 사용자 집계(사람수·기록수·이번달 챙김), 성향 4축 점수, 감정 빈도 집계, 감사일기 최근 1건.

**AC**
- [ ] 검정 히어로 카드에 성향 한 문장이 표시된다. *(성향 산출 규칙 → O-3)*
- [ ] 스탯 3칸·자주 느낀 감정이 샘플 데이터로 채워진다.
- [ ] 4축 바(P2 미구현 시)는 정적 값으로 표시 가능.

## 7. 데이터 모델 (개념 수준)

> 데모는 **샘플 데이터 + 로컬 저장**으로도 성립. 아래는 개념 스키마(정식 설계는 개발이 확정).

- **User(나)** — 이름, 이니셜, 가입시점(관계도감 N년차)
- **Person(인물)** — 이름, 이니셜, 즐겨찾기, 태그[], 생일, 첫만남일, 마지막연락(일자·채널), 친밀도등급, 좋아하는것[], 조심할것[]
- **Event(기록)** — 대상Person, 일자, 카테고리(만남/연락/기념일/프로젝트/기타), 제목, 감정[], 날씨, 함께한사람[], 사진[], 언제/왜/무엇, 본문(선택), 마커메모(선택)
- **파생/집계** — 인물별 만남수·마지막만남·기록수, 월별 만남/연락/추억, 사용자 성향 4축·감정 빈도, "1년 전 오늘" 매칭, 공백 구간

## 8. 비기능 요구사항
- **플랫폼:** 모바일 우선(세로), 데모는 모바일 웹/앱 형태.
- **성능:** 타임라인 이벤트 다수 시 가상 스크롤 고려(데모는 샘플 수준이라 후순위).
- **오프라인/저장:** 데모는 샘플 시드 데이터 기반, 기록 저장은 세션/로컬 수준으로 충분.
- **접근성:** B&W 고대비이므로 명도 대비는 유리. 칩·탭 터치 타깃 충분히.
- **개인정보:** 사적 도감 컨셉 — 기본 "나만 보기". (공유·범위 설정 UI는 이번 스코프 밖.)

## 9. 범위 (In / Out) 요약

**In (이번에 만든다 — 스케치에 있음)**
관계 지도(홈) · 즐겨찾기/사람추가 · 1년 전 오늘 플로팅 · 사람 프로필(태그·스탯·기본정보·최근일·선호/비선호) · 사람별 타임라인(이벤트 피드·구조 필드) · 칩 기반 기록 작성 · 나 인사이트(성향 한 줄) · 하단 탭.
P2: 활동 흐름 차트 · 성향 4축 바 · 감정 통계. P3: 리스트 토글·날씨·마커·감사일기·편집·미니 일러스트·본문 장문.

**Out (이번 범위 밖 — 스펙아웃 / mock 「준비중」)**
- 온보딩 자동 수집(캘린더·갤러리·카톡·Slack) — *화면 없음·기술 벽*
- AI 자동 파악/요약, 제3자 AI 점검, 감정 정성 분석 — *화면 없음*
- 관계 온도 게이지/곡선 계산·온도 서사 — *최종 화면에서 빠짐(단, S3 "활동 흐름"은 유지)*
- 알림·넛지 푸시 시스템(뜸한 사람 알림·정리 제안·기념일 푸시) — *플로팅 회고만 있고 푸시 없음*
- 공유·공동 기억·팔로우·공개 범위 설정 — *화면 없음·양면 시장*
- 선물 추천·센스 보조·두 사람 이어주기 — *화면 없음*
- 그림일기(AI 그림·원고지) — *칩 기반으로 확정*
- 통합/맵/인생 타임라인·자동 회고·추억 카드·나만의 책 — *화면 없음*
- 로그인·회원가입 — *데모 불필요 → mock*

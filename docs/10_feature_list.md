# Step 10. 기능 나열

> 목적: 지금까지 논의에서 나온 **기능 아이디어를 하나도 빠뜨리지 않고** 모은다.

> **Guide:**  
> 1. Agent가 `docs/04`~`docs/09`을 읽고, 언급된 **모든 기능**을 추출한다.  
> 2. **순서·중요도·선후 관계 없이** 일렬로 나열한다.  
> 3. 팀이 빠진 기능이 있으면 **팀이 추가한 기능**에 적는다.  
> - 기능은 **사용자가 할 수 있는 일** 또는 **서비스가 제공하는 것** 단위로 쓴다. (예: 로그인, 일정 자동 추천, 알림 보내기)  
> - 큰 덩어리(예: "앱 전체")로만 적혀 있어도 **일단 포함**하고, 안에 여러 기능이 섞여 있으면 **풀어서** 각각 적는다.  
> - 비슷한 기능이라도 **논의에서 따로 나왔으면 각각** 적는다.  
> - **이 단계에서 하지 않는 것:** 우선순위 정하기, 중요도 판단, MVP만 골라내기, 제외·합치기  
> - 우선순위·제외는 **Step 14**에서 한다. 여기서는 **놓치지 않고 모으는 것**만 한다.

**프롬프트 예시**

```
docs/04~09을 읽고 docs/10_feature_list.md의 "Agent가 추출한 기능" 섹션만 채워줘.

규칙:
- docs/04~09 어디에든 나온 기능·아이디어를 빠짐없이 포함 (목적·가치, 여정, Pain/Wow, 가설, 키워드, 질문·답변 등)
- F-01부터 번호를 매겨 일렬로 나열 (번호·순서는 우선순위가 아님)
- 기능명은 짧고 구체적으로 (한 줄)
- 우선순위 판단·제외·MVP 선별 하지 않는다. 의심스러워도 일단 나열
- "팀이 추가한 기능" 섹션은 건드리지 않는다.
- 구조와 상단 Guide는 유지
```

## Agent가 추출한 기능

> `docs/04`~`docs/09` 및 Day1 FigJam 브레인스토밍에서 나온 기능. **빠짐없이** 나열. (번호·순서·묶음은 우선순위가 아님. 우선순위·제외는 Step 14에서.)

**A. 기록·입력 (부담 없이 빠르게)**
- **F-01** 감정 chip 선택으로 30초 안에 빠른 기록
- **F-02** 세 줄 이내 짧은 글 기록 (글자 수 제한으로 압축 유도)
- **F-03** 장문(줄글) 기록 — 추가 옵션
- **F-04** 그림일기 형식으로 기록
- **F-05** 만남 직후 그날의 감정·이야기 즉시 기록
- **F-06** 한 인물에 사진 연결
- **F-07** 한 인물에 메모 연결
- **F-08** 만남·연락 이벤트를 일자별로 기록
- **F-09** 그 사람의 선호/비선호 기록
- **F-10** 나의 감정·해석 직접 입력
- **F-11** 스토리 태그로 기록
- **F-12** 기록 시간대 알림 설정 (하루 회고 리마인드)
- **F-13** GPS 기반 현재 위치 자동 입력

**B. 자동 수집·연동 (기록 최소화)**
- **F-14** 캘린더 일정 자동 수집
- **F-15** 사진 갤러리 날짜 자동 수집
- **F-16** 카카오톡 대화내역 불러오기 (import)
- **F-17** Slack MCP/API 연동 자동 입력
- **F-18** 흩어진 관계 흔적을 사람 중심으로 자동 통합
- **F-19** AI가 나 대신 관계를 자동 파악

**C. 사람 중심 조회**
- **F-20** 인물 페이지 — 한 사람의 만남·연락·사진·기념일·메모를 한눈에
- **F-21** 첫 화면 인물별 대시보드
- **F-22** 인물 클릭 시 만난 일자·날짜를 카드·목록으로 나열
- **F-23** 인물별 사진첩 형태로 추억 보기
- **F-24** 주요 인물 우선 노출 (가장 오래된 / 새로 추가된 / 중요한 관계)

**D. 시각화·관계도**
- **F-25** 나를 중심으로 한 관계도(살아있는 지도·인물관계도) 시각화
- **F-26** 관계 온도 표시
- **F-27** 관계의 흐름(시간에 따른 가까워짐/멀어짐) 시각화
- **F-28** 만남 빈도수 시각화
- **F-29** 기준(물질적/감정적 등)에 따라 관계 뷰 전환

**E. 타임라인·회고**
- **F-30** 사람별 타임라인
- **F-31** 전체 인물 통합 타임라인 (구간별)
- **F-32** 타임라인 맵
- **F-33** 인생 타임라인 (데이터 누적 시)
- **F-34** 사람 중심 자동 회고 (애플 사진첩식이되 사진이 아니라 사람 기준)
- **F-35** "1년 전 오늘" · "작년 이맘때 이 사람과" 회고 표시

**F. 알림·넛지·재연락**
- **F-36** 연락 뜸해진 인물 푸시 알림
- **F-37** 연락·만남 주기 분석
- **F-38** 비활성 관계 점검 (긴 주기로 만나는 친구 등)
- **F-39** "오랫동안 연락 안 한 사람, 정리할까요?" 제안
- **F-40** 생일·기념일 다가옴 알림 (관계 맥락 포함)
- **F-41** "이맘때 만나서 뭐 했어요" 상기
- **F-42** 만났던 사람을 잊지 않았는지 alert

**G. 분석·인사이트 (AI)**
- **F-43** 내 관계 성향·나에 대한 이해 자동 분석 (나의 성향 파악)
- **F-44** 관계를 제3자(AI) 관점에서 점검
- **F-45** 감정 분석 (정성)
- **F-46** 상호작용·빈도 집계 (정량)
- **F-47** AI 관계 요약 (일일이 다시보기 대신 요약)
- **F-48** 주간 관계 리포트
- **F-49** 감사일기 (고마운 사람·일 기록)

**H. 추억 아티팩트**
- **F-50** 추억 카드 (사진 + 날짜 + 짧은 문장 묶음)
- **F-51** 관계의 온도·서사를 함께 담은 기록 (단순 만남 로그가 아님)
- **F-52** 기록이 모여 나만의 책·아카이브(전자 일기장)가 됨

**I. 공유·공동 기억**
- **F-53** 친한 사람을 초대해 같은 기록 공간 공유
- **F-54** 같은 추억을 서로의 시점에서 연결해 보기 (공동 기억)
- **F-55** 서로 팔로우
- **F-56** 일기·기록 선택적 공유

**J. 프라이버시**
- **F-57** 프라이빗 vs 공유 범위 구분 설정

**K. 챙김 액션**
- **F-58** 선호/비선호 기반 선물 추천
- **F-59** 눈치·센스 보조 (다음 행동 제안)
- **F-60** 두 사람 이어주기 추천 (네트워킹 자산화) — *논의에서 나옴, Step 14에서 취사 결정*

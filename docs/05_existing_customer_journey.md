# Step 05. 기존 고객 여정

> 목적: `docs/04`에서 정의한 서비스 목적을, **지금** 사용자가 어떤 방식으로 달성하는지 정리한다.

> **Guide:** **우리가 만들 서비스의 여정이 아니다.** (우리 서비스 여정은 Step 12 `docs/12_customer_journey_map.md`) 유사 서비스·대체제·우회 방식마다 **여정을 하나씩** 두고, 단계별 사용자 행동·생각/감정을 표로 정리한다. 각 단계는 상세하게 작성한다. 이 내용을 토대로 Step 06에서 Pain/Wow 인사이트를 추출한다.

## 여정 1

> 카카오톡 + 캘린더 + 사진첩 — 여러 앱에 흩어진 조각을 감으로 짜맞추기

| 단계 | 사용자 행동 | 생각/감정 |
|---|---|---|
| 관계 시작 | 카카오톡으로 친구 추가 | "일단 연락처는 카톡에" |
| 약속 | 카톡으로 약속을 잡음 | 대화는 카톡에 흩어짐 |
| 일정화 | 캘린더에 일정 저장 | "약속은 잊으면 안 되니까" |
| 만남 | 만났을 때 사진을 찍음 | 순간을 사진첩에 남김 |
| 챙김 | 카톡 생일 알림으로 챙김 | 형식적인 계기 하나뿐 |
| 회상 | 나중에 카톡·사진첩·캘린더를 각각 뒤져 추억을 찾음 | "그때 그 사람 어땠더라" — 비효율·불편 |
| 결과 | 조각이 사람 중심으로 연결 안 됨 | 관계 흐름·맥락·감정이 사라짐 |

## 여정 2

> 그냥 기억에 맡기기 (별도 도구 없음)

| 단계 | 사용자 행동 | 생각/감정 |
|---|---|---|
| 유지 | 관계를 순전히 기억에만 의존 | 친밀하면 기억, 중요해도 안 친밀하면 놓침 |
| 경과 | 시간이 지나며 소원해진 사람은 자연히 잊힘 | "사라지니 아쉽지만… 그렇게 흘러감" |
| 재회 전 | 막연한 느낌으로만 정리된 상태 | "그때 친했는데 왜 가까웠는지" 단서가 없음 |
| 재회 시 | 다시 만나 얘기하면 되살아남 | 그 전까지는 사실상 사라진 상태 |

## 여정 3

> 개인 일기 / 메모앱에 직접 기록

| 단계 | 사용자 행동 | 생각/감정 |
|---|---|---|
| 기록 | 일기·메모앱에 직접 작성 | 직접 써야 해서 귀찮음 (입력이 부담) |
| 지속 | 일기 자체를 잘 안 씀 | 습관이 안 됨, 꾸준함이 어려움 |
| 회상 | 나중에 다시 봄 | 다시 보기 쉬운 구조가 아님 |
| 한계 | 정보가 구체적이지 않게 적혀 있음 | 그때의 감정·생각이 안 남아 온전히 추억하기 힘듦 |

## 여정 4

> 사진첩 / SNS(인스타그램 스토리·DM)로 추억하기

| 단계 | 사용자 행동 | 생각/감정 |
|---|---|---|
| 사진첩 | 사진첩으로만 추억을 되짚음 | 순간은 있으나 **사람별 맥락**이 없음 |
| SNS 근황 | 인스타 스토리·DM으로 근황 확인, "1년 전 스토리" 보기 | 근황은 알지만… |
| SNS 한계 | 점점 남에게 보여주기 위한 자랑·광고 위주로 흐름 | 오히려 나를 숨기게 만듦, 사적 관계 기록엔 부적합 |

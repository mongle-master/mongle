# Step 11. 기능을 페이지로 나누기

> 목적: Step 10에서 모은 **기능을 화면(페이지) 단위로 나눠** 배치한다.

> **Guide:**  
> 1. Agent가 `docs/10_feature_list.md`의 기능 목록을 읽고 **페이지 구성 초안**을 작성한다.  
> 2. 각 페이지에 **어떤 기능(F-xx)** 이 들어가는지 나열한다.  
> 3. 팀이 페이지 수·이름·구성을 보며 수정한다.  
> - Step 10의 기능은 **어느 페이지에든 모두 배치**한다. (이 단계에서 기능을 빼지 않는다)  
> - 사용자 흐름이 자연스럽게 이어지도록 페이지를 묶는다. (예: 로그인 → 메인 → 상세)  
> - 한 페이지에 기능이 너무 많으면 **페이지를 나눈다.**  
> - **우선순위·스펙 아웃(이번에 만들지 않을 기능)** 은 Step 14 `docs/14_priority_matrix.md`에서 정한다.

**프롬프트 예시**

```
docs/10_feature_list.md와 docs/04~09 맥락을 읽고 docs/11_page_routing_split.md를 채워줘.

규칙:
- Step 10의 모든 기능(F-xx)을 어느 페이지에든 배치 (누락 없음)
- 페이지마다 이름·경로·목적·포함 기능 목록을 작성
- 기능 ID(F-xx)와 기능명을 함께 적기
- 해커톤 MVP 기준으로 페이지 수는 과하지 않게 (보통 3~6개)
- "페이지 간 이동 흐름" 섹션도 채우기
- 구조와 상단 Guide는 유지
```

> **배치 원칙:** F-01~60 전부를 6개 페이지에 **누락 없이** 배치. 넛지·공유는 별도 페이지 대신 관련 페이지의 **섹션**으로 접음. (제외·우선순위는 Step 14에서)

## 페이지 1: 온보딩 · 연동 · 설정

> **경로:** `/onboarding`, `/settings`  
> **목적:** 따뜻한 첫인상으로 시작해, 흩어진 관계 흔적을 **자동으로 끌어오게** 연결하고 공개 범위를 정한다. (기록 부담 최소화의 출발점)

- **F-14** 캘린더 일정 자동 수집
- **F-15** 사진 갤러리 날짜 자동 수집
- **F-16** 카카오톡 대화내역 불러오기(import)
- **F-17** Slack MCP/API 연동 자동 입력
- **F-18** 흩어진 관계 흔적을 사람 중심으로 자동 통합
- **F-19** AI가 나 대신 관계를 자동 파악
- **F-12** 기록 시간대 알림 설정(하루 회고 리마인드)
- **F-57** 프라이빗 vs 공유 범위 구분 설정

## 페이지 2: 홈 — 관계 지도 & 챙길 거리

> **경로:** `/`  
> **목적:** 나를 중심으로 한 **살아있는 관계 지도**를 한눈에 보여주고, "지금 챙길 사람"을 표면에 띄운다. (자주 열어보게 만드는 메인)

- **F-21** 첫 화면 인물별 대시보드
- **F-24** 주요 인물 우선 노출(오래된/신규/중요)
- **F-25** 나 중심 관계도(살아있는 지도·인물관계도) 시각화
- **F-26** 관계 온도 표시
- **F-27** 관계의 흐름(가까워짐/멀어짐) 시각화
- **F-28** 만남 빈도수 시각화
- **F-29** 기준(물질적/감정적)별 관계 뷰 전환
- _섹션: 오늘 챙길 사람 (넛지)_
  - **F-36** 연락 뜸해진 인물 푸시 알림
  - **F-37** 연락·만남 주기 분석
  - **F-38** 비활성 관계 점검(긴 주기 친구)
  - **F-39** "오래 연락 안 한 사람, 정리할까요?" 제안
  - **F-40** 생일·기념일 다가옴 알림(관계 맥락 포함)
  - **F-42** 만났던 사람을 잊지 않았는지 alert

- **F-20** 인물 페이지(만남·연락·사진·기념일·메모 한눈에)

- **F-30** 사람별 타임라인

- **F-51** 관계의 온도·서사를 담은 기록(만남 로그가 아님)

- **F-04** 그림일기 형식 기록
- **F-05** 만남 직후 감정·이야기 즉시 기록
- **F-06** 사진 연결
- **F-07** 메모 연결
- **F-08** 만남·연락 이벤트 일자 기록
- **F-10** 나의 감정·해석 직접 입력

- **F-43** 내 관계 성향·나에 대한 이해 자동 분석

## 페이지 간 이동 흐름

> 사용자가 페이지를 어떻게 오가는지 순서대로 적는다.

1. **첫 실행** → `온보딩(P1)`: 따뜻한 소개를 보고 카톡·캘린더·사진 연동 + 공개 범위 설정 → 자동 수집으로 초기 관계가 채워짐
2. **매일/주기적 진입** → `홈(P2)`: 관계 지도를 열고 "오늘 챙길 사람" 넛지를 확인
3. 관심 가는 인물 클릭 → `인물 상세(P3)`: 그 사람과의 흐름·추억을 보고, 선물 추천·다음 행동 참고, 원하면 초대·팔로우
4. **만남·연락 직후** → `기록하기(P4)`: chip·세 줄로 30초 기록 (또는 자동 수집이 대신 채움)
5. 여유 있을 때 → `타임라인(P5)`: 전체 회고, "1년 전 오늘"·추억 카드로 잊은 인연 재발견 → 다시 P3로 들어가 챙김
6. 주간 단위 → `인사이트(P6)`: 나의 관계 성향·주간 리포트·감사일기로 나를 돌아봄

# Step 13. 스케치 및 결정

> 목적: 화면 설계안을 **각자 그려본 뒤** 팀이 맞추고, **최종 스케치만** 남긴다.

> **Guide:** (Step 02처럼 **대면 워크숍**으로 진행. Agent가 대신 채우지 않는다.)  
> 1. **각자** AI(v0, Bolt.new, Cursor 등)에게 `docs/11`·`docs/12`을 참고해 **페이지별 화면 스케치**를 그려본다.  
> 2. 팀이 스케치를 **공유**하고, 레이아웃·흐름·느낌을 **이야기**한다.  
> 3. 합의된 **최종 스케치**만 이 문서에 등록한다.  
> - 스케치 생성·비교는 각자·팀에서 한다. 이 문서에는 **결정 결과**만 적는다.  
> - 여러 후보 중 **팀이 고른 것**만 적는다. 개인 스케치 전부를 문서에 옮기지 않는다.

> ⚠️ **아래는 팀 확정 전 "초안(제안)"** — 스프린트 마스터가 그린 출발점. 팀이 워크숍에서 보고 수정·확정한다.

## 컨셉: 연대기 (Chronicle) — 시간이 주인공

> 남들의 "관계 트래커"는 대부분 **관계 그래프 + 인물 프로필 + 기록 폼**으로 수렴한다(Dex·Clay·YourPond). 겹치지 않기 위해, 관계도감은 **시간축(연대기)을 앱의 뼈대**로 삼는다.
> - 세로로 흐르는 **시간**이 메인. 사람은 그 위를 흐른다.
> - 멀어진 관계 = 연대기의 **공백(점선 구간)**으로 시각화. (남들은 "연락하세요" 알림, 우리는 *빈 자리 자체*를 보여줌)
> - 인물 = 프로필이 아니라 **관계 온도가 오르내린 이야기 곡선**.
> - 기록 = 폼이 아니라 **끊긴 이야기를 잇는 행위**("연대기에 새기기"). → Step 07의 "왜 기록하나" 동기 문제를 서사로 해결.

## 최종 스케치

> 페이지별로 팀이 선택한 스케치.

### 연대기 홈 (docs/11 P2 재해석)

> **선택:** _초안 (스프린트 마스터 제안) — 팀 확정 대기_  
> **링크·이미지:** `docs/assets/sketch/mock-home.png`  
> **한 줄 요약:** 세로 시간축 **피드**. 각 카드 = 하나의 만남/사건 — 타임라인 **아이콘 뱃지**(여행·국밥·모임…) + 제목 + **카테고리 태그**(만남/연락/기념일/기타) + **누구랑 / 왜 만났는지 / 무엇을 했는지** + **얇은 라인 일러스트**. 상단 필터 칩, 하단 탭(홈·사람·기록·보관함) + FAB. 특별한 메모는 손그림 **마커 하이라이트**.  
> **무드:** "심플하지만 그림일기" — 정갈한 카드 + 세련된 라인 일러스트 + 최소 파스텔 악센트(블롭·마커). (레퍼런스 반영)  
> **유지:** 멀어진 관계의 **공백 시각화**(예: "민지와 3개월째 조용해요")는 피드 안 **특별 카드**로 살린다. (이 목업엔 미표시)

### 인물의 연대기 (docs/11 P3 재해석)

> **선택:** _초안 (스프린트 마스터 제안) — 팀 확정 대기_  
> **링크·이미지:** `docs/assets/sketch/mock-person.png`  
> **한 줄 요약:** 상단에 **관계 온도 흐름 곡선**(절정 → 지금 하락, 4년의 이야기). 아래 "우리의 연대기" 타임라인은 3개월 공백 pill → 벚꽃놀이 → 국밥 → 처음 만난 날(시작점). 하단 "먼저 잇기 / +오늘 새기기".

### 오늘 새기기 — 그림일기 (docs/11 P4 재해석)

> **선택:** _초안 (스프린트 마스터 제안) — 팀 확정 대기_  
> **링크·이미지:** `docs/assets/sketch/mock-record.png`  
> **한 줄 요약:** 오늘을 **정통 그림일기장** 포맷으로 남긴다 — ① 날짜·날씨 표 칸(테두리 셀, 날씨 선택) ② 그림 칸(**AI가 B&W 스케치로 변환**, "참 잘했어요" 스탬프) ③ **제목 칸 + 오늘의 낱말 태그(톡 눌러 선택 + ＋낱말)** ④ **원고지 격자(한 글자씩)**. 입력은 낱말·감정 칩만(30초), 그림·글은 AI가 채움. "직접 그리기" 옵션. 저장: "연대기에 새기기 · 3개월 공백을 그림 한 장으로". (레퍼런스: 초등 그림일기장)  
> **결정 근거:** 그림일기(아기자기·따뜻 = docs/08 키워드)를 살리되, **직접 그리기를 강제하면 "부담 없는 기록(30초)" 기둥과 Step 07 최대 리스크가 깨짐.** 그래서 **AI 생성**을 기본, 손그림은 옵션으로. → 감성 ↑, 부담 그대로.  
> **낱말 태그:** 타이핑 대신 **톡 눌러 고르는** 최소 입력 수단. 동시에 나중에 **사람별 낱말 구름·검색·회고**의 데이터가 됨(입력이자 자산). (F-11 스토리 태그 확장)

### 나 · 인사이트 (docs/11 P6)

> **선택:** _초안 (스프린트 마스터 제안) — 팀 확정 대기_  
> **링크·이미지:** `docs/assets/sketch/mock-me.png`  
> **한 줄 요약:** "관계도감으로 **나를 알아가기**"를 담는 화면. 검정 히어로에 관계 데이터가 규정한 한 문장("당신은 오래된 관계를 오래 지키는 사람") + 성향 4축 바 + 자주 느낀 감정 + 주간 감사일기. (F-43·45·46·48·49)

## 합의 사항

> 스케치 토론에서 팀이 맞춘 내용

- **디자인 방향:** **연대기(시간 주인공) × 블랙&화이트 에디토리얼.** 유채색 0, 흑·백·회색 + 타이포·여백·둥근 형태. "따뜻함·힐링"은 색 대신 넉넉한 여백·친근한 카피·모노그램으로. SNS의 요란함과 반대되는 담백·사적인 도감.
- **공통 UI 규칙 (초안):**
  - 색 — 배경 화이트 `#ffffff` / 패널 `#fafafa`·플레이스홀더 `#f2f2f2` / 테두리 `#e6e6e6` / 잉크(강조·버튼) `#1c1c1c` / 서브 `#8c8c8c`·흐림 `#b5b5b5`. 유채색은 거의 0 — **단, 레퍼런스처럼 마커 하이라이트(연녹)·모서리 파스텔 블롭 같은 최소 악센트만 허용.**
  - 일러스트 — **얇은 라인(1.5px) 그림일기 삽화.** 스틱맨 X, 과한 채색 X. 사건 성격(만남·식사·여행·모임)에 맞는 아이콘 뱃지 + 삽화.
  - 시간축 — 세로 스파인(2px `#e4e4e4`), 노드 채움 `#141414`(기록됨)·비움(공백/시작점). **공백은 점선 스파인**으로.
  - 대비 — 선택·활성 = 검정 채움(흰 글씨), 비활성 = 흰 배경·회색 테두리. 강조 단어는 밑줄.
  - 아바타 = **모노그램(이니셜)**. 사진·아이콘 자리는 회색 플레이스홀더.
  - 내비 — 하단 탭(연대기·사람·인사이트·설정) + 중앙 검정 `＋`.
- **구현 시 유의:**
  - **"공백" 판정 로직** — 얼마나 비면 공백인가? 사람마다 만남 주기가 다르니 개인화 필요(고정 임계값 X). ← 데모 임팩트의 핵심.
  - **관계 온도 곡선을 뭘로 계산하나** — 만남 빈도·기록 감정 기반. Step 07 Wow4 가설("자동 수집으로 진짜 온도가 나오나")과 직결. 데모는 규칙 기반으로 시작 가능.
  - **관계 지도(F-25)** 는 이제 메인에서 빠짐 → **보조 뷰 or 스펙아웃 후보**로 Step 14에서 판정.
  - **콜드 스타트** — 온보딩 자동 수집이 없으면 연대기가 텅 비어 첫인상이 약함. 여전히 최우선 의존.
  - **연대기 스크롤** — 기록 많아지면 가상 스크롤 필요(성능).

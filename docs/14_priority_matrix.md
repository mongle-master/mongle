# Step 14. 기능 우선순위 분류 (임팩트/시간)

> 목적: `docs/11`의 **페이지별 기능(F-xx)** 마다 **1~4순위**를 정하고, 무엇을 만들고·버릴지·mock 할지 확정한다.

> **Guide:**  
> 1. `docs/11_page_routing_split.md`의 **페이지별 기능 목록**을 본다.  
> 2. **기능 하나마다** 아래 4가지 중 하나에 넣는다. (페이지 단위가 아니라 **기능 단위**)  
> 3. 팀이 토론하며 **1순위 = 이번에 꼭 구현**할 것을 맞춘다.  
> - **1순위** — 높은 임팩트 / 짧은 구현 시간 (즉시 개발)  
> - **2순위** — 높은 임팩트 / 긴 구현 시간 (핵심 확장)  
> - **3순위** — 낮은 임팩트 / 짧은 구현 시간 (여유 있으면)  
> - **4순위** — 낮은 임팩트 / 긴 구현 시간 (이번 범위 **스펙 아웃**·mock 후보)

> ⭐ **이번 분류 기준 = 확정 화면(`docs/assets/sketch/` 5장).**
> **화면에 그려진 것 = 만든다. 화면에 없는 것 = 스펙아웃.** (docs/10 백로그가 아니라 화면이 기준)

**Tip — 읽어보기**

- 핵심 기능만 제대로. 핵심 아닌 건 과감히 버리거나 **동작 안 하는 mock**.
- *로그인·회원가입* 은 데모 필수 아님 → mock/생략.
- 미구현 버튼은 **「준비중입니다」** 표시 (아무 반응 없으면 버그처럼 보임).

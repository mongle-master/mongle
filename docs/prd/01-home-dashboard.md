# 01. 홈 · 관계 대시보드

> **SSOT:** 홈 화면(그래프/리스트뷰, 태그 필터, 1년 전 오늘 플로팅, 하단 내비게이션)의 단일 기준.
> 상위 배경·목표: [개요](./README.md) · 관련 화면: [02 사람](./02-person.md), [03 타임라인](./03-timeline.md), [04 기록 작성](./04-record.md)
> 참고: 즐겨찾기 **토글 정의**는 [02 사람](./02-person.md#5-즐겨찾기), 이 문서는 관계도 맵에서의 **표시** 만 다룬다.
> 공통 정의(내비·칩·날짜·표시 포맷·문구·이미지·접근성): [공통 기준](./README.md#12-공통-기준-전-화면-공유) — 각 항목은 여기서 재정의하지 않고 참조한다.
> 백엔드: [홈(home) 이슈](https://github.com/mongle-master/mongle/labels/home)

## 1. 목적

사용자가 전체 관계를 한눈에 보고, 사람 추가와 타임라인 진입을 빠르게 수행한다.

## 2. 화면 요소

- 상단 보기 전환: 그래프뷰 / 리스트뷰
- 관계도 또는 리스팅 선택 UI
- 관계 그래프 또는 사람 리스트
- 타임라인으로 이동하는 진입점 → [03](./03-timeline.md)
- `1년 전 오늘 누구와 만났는가` 플로팅 (조건부 노출, [§5](#5-정책) 참조)
- 사람 추가 버튼 → [02 사람 등록](./02-person.md#2-사람-추가--프로필-등록)
- 하단 내비게이션: 홈 · 타임라인 · ＋(기록) · 사람 · 설정 (구성·라벨 정의 → [공통 기준](./README.md#12-공통-기준-전-화면-공유))
- 태그 chip 필터 및 chip 제거 기능

## 3. 상호작용

- 그래프뷰를 선택하면 인물 노드(프로필 이미지 + 이름)와 관계 태그를 시각적으로 보여준다.
- 리스트뷰를 선택하면 사람 카드 목록을 보여준다.
- 즐겨찾기한 사람은 그래프뷰에서 프로필 테두리를 강조한다. (토글 정의 → [02](./02-person.md#5-즐겨찾기))
- `1년 전 오늘` 플로팅을 누르면 해당 기록 상세 또는 타임라인으로 이동한다. → [03](./03-timeline.md)
- `+ 사람 추가`를 누르면 프로필 등록 화면으로 이동한다. → [02](./02-person.md#2-사람-추가--프로필-등록)
- 태그 chip을 누르면 해당 태그의 인물만 필터링, chip의 제거(×)로 필터 해제한다.

## 4. 입력·필드 상세 스펙

> 홈은 입력보다 보기·탐색 중심 화면이라, 아래는 화면의 컨트롤과 표시 규칙을 정리한 것이다. (글자수·날짜 등 공통 규칙은 [공통 기준](./README.md#12-공통-기준-전-화면-공유))

| 항목 | 값/타입 | 제약·규칙 | 기본값 | 비고 |
|---|---|---|---|---|
| 기본 보기 모드 | 그래프뷰 / 리스트뷰 전환 | 마지막에 고른 보기를 기억해 다음에 그대로 보여준다 | 그래프뷰(처음 진입) | 홈은 관계 지도가 메인 |
| 그래프 노드 표시 | 노드 안=**프로필 이미지**(없으면 이름 첫 글자) / 노드 아래=이름 | 이미지와 이름을 함께 보여준다. 이름이 길면 관계도에서는 줄여서(…) 보이고, 전체 이름은 그 사람을 눌러 확인 | 이미지+이름 | 사진 없을 때만 이름 첫 글자(공통 아바타 규칙) |
| 태그 필터 | 관계태그 다중 선택 | 여러 태그를 함께 고르면 그중 하나라도 가진 사람을 모두 보인다(OR). 고른 태그는 ×로 개별 해제, `필터 초기화`로 전체 해제 | 미선택(=전체) | 관계태그가 하나도 없으면 필터 영역을 보이지 않음 |
| 리스트 정렬(홈 리스트뷰) | 고정 | 즐겨찾기 상단 → 최근 만난 순 → 이름 가나다순 | (고정, 사용자 조절 없음) | 홈 리스트뷰는 개요용. 정렬·검색은 사람 탭에서 |
| 정렬(사람 탭) | 이름순 / 최근 만남순 (+즐겨찾기 상단) | 이름순은 가나다(한글→영문→숫자, 대소문자 무시) | 이름순 + 즐겨찾기 상단(켬) | 상세 → [§10](#10-사람-탭-인물-디렉토리) |
| 검색(사람 탭) | 텍스트 | 이름·관계 유형으로 부분 일치(대소문자·공백 무시) | 빈 값 | 초성 검색은 `[post-MVP]` |
| 1년 전 오늘 플로팅 | 조건부 카드 | 정확히 1년 전 기록이 있을 때만. 여러 건이면 우선순위로 1장만 | 조건 충족 시 노출 | 우선순위·이동·닫기 → [§5](#5-정책) |
| 인물 0명일 때 | 빈 상태 안내 | 사람 추가 유도 | — | 문구 → [§6](#6-상태엣지케이스) |

## 5. 정책

- **기본 보기:** 홈은 관계 지도가 메인이므로 처음엔 **그래프뷰**로 연다. 이후에는 사용자가 마지막에 고른 보기(그래프/리스트)를 기억해 그대로 보여준다.

- **1년 전 오늘 플로팅:** 오늘로부터 **정확히 1년 전(같은 월·일)** 기록이 있을 때만 노출한다. 없으면 띄우지 않는다. (2월 29일은 해당 연도에 없으면 노출하지 않음)
  - **여러 건일 때는 한 번에 한 장만** 보이며, 다음 우선순위로 고른다: ① 즐겨찾기한 사람과의 기록 → ② 사진이 있는 기록 → ③ 카테고리가 `기념일`인 기록 → ④ 그날 기록 시간이 이른 것 → 기록을 남긴 순.
  - **누르면** 그 사람의 타임라인으로 가서 해당 기록을 펼쳐(강조해) 보여준다. → [03](./03-timeline.md)
  - **×로 닫으면 그날 하루는 다시 띄우지 않는다**(기기에 기억). 날짜가 바뀌면 조건을 다시 확인해 노출한다.
  - 카드 문구: `1년 전 오늘, {이름}님과` + `{기록 제목}, 기억나요?` (제목이 없으면 `함께한 순간, 기억나요?`).

- **즐겨찾기 표시:** 즐겨찾기한 사람은 관계도에서 **테두리를 강조하고 ★로 표시**한다. 리스트·사람 탭 목록에서는 **이름 옆 ★와 함께 위쪽에 모아** 보여준다. (토글 동작은 [02](./02-person.md#5-즐겨찾기))

- **멀어진 관계(흐린 표시):** 어떤 사람과 **평소 만남 주기보다 한참 뜸해지면** 관계도에서 그 사람의 연결선과 이름을 흐리게 보여준다. (얼마나 뜸해야 흐리게 볼지 구체 기준은 구현 단계에서 정하고 조정한다.)
  - 사람마다 만나는 주기가 다르므로 고정된 기간이 아니라 그 사람의 **평소 주기**를 기준으로 삼는다. 아직 만남이 거의 없어 주기를 알 수 없는 사람은 흐리게 하지 않는다.
  - 즐겨찾기한 사람도 대상이다 — 멀어진 소중한 사람을 다시 떠올리게 하려는 것이다. 이때도 즐겨찾기 강조(테두리·★)는 유지한다.

- **태그 chip 필터·제거:** 관계태그로 사람을 걸러 본다.
  - 여러 태그를 함께 고르면 **그중 하나라도 가진 사람**을 모두 보인다(OR·합집합). 좁히기보다 "친구 또는 동료 함께 보기"처럼 **넓혀 보는** 의도에 맞춘다.
  - 고른 태그는 **×로 하나씩** 해제하고, 하나 이상 골랐을 때 **`필터 초기화`**로 한 번에 해제한다.
  - 그래프뷰에서 조건에 맞지 않는 사람은 **숨긴다**(흐린 표시는 `멀어진 관계` 전용이라 섞지 않는다).

- 관계를 점수화·서열화하는 표현은 사용하지 않는다.

## 6. 상태·엣지케이스

> 문구는 공통 마이크로카피를 재사용한다. ([공통 기준](./README.md#12-공통-기준-전-화면-공유))

- **인물 0명(콜드스타트):** 관계도에는 `나`와 `＋ 사람 추가`만 두고 안내를 보인다. 태그 필터·회고 플로팅은 보이지 않는다.
  - `"아직 기록한 사람이 없어요. 첫 사람을 추가해 관계를 남겨보세요."` + `[＋ 사람 추가]`
- **관계태그가 하나도 없을 때:** 필터 영역을 보이지 않는다(빈 필터 줄을 만들지 않는다). 관계태그는 기본 제공값이 없어 처음엔 비어 있는 게 정상이다.
- **필터 결과 0명:** 필터는 유지하고 결과 영역만 빈 상태로 둔다.
  - `"이 조건에 맞는 사람이 없어요."` + `[필터 초기화]`
- **관계도 과밀(사람이 많을 때):** 사람이 많아 관계도로 한눈에 보기 어려우면 리스트 보기를 권한다: `"인원이 많아요. 리스트로 보면 한눈에 편해요."` + `[리스트로 보기]`
- **관계 지도 불러오기 실패:** `"관계 지도를 불러오지 못했어요. 잠시 후 다시 시도해 주세요."` + `[다시 시도]`
- **1년 전 오늘 기록 없음:** 플로팅 미노출 (확정)

## 8. 관련 사용자 흐름

**관계 대시보드에서 사람을 찾는다**
1. 홈에서 그래프뷰 또는 리스트뷰를 선택한다.
2. 태그 chip으로 관계를 필터링한다.
3. 사람 노드 또는 사람 카드를 누른다.
4. 사람 프로필에서 최근 함께한 일과 관계 요약을 확인한다. → [02](./02-person.md)

## 9. 열린 질문 / 리스크

- **(결정됨) 하단 내비게이션 = 홈 · 타임라인 · ＋(기록) · 사람 · 설정.** 인사이트 탭 제외. `타임라인`은 전 인물 기록을 시간순으로 보는 나의 통합 연대기(전역), 인물별 타임라인은 사람 프로필 안의 탭이다. 근거: 관계 지도가 메인(홈)이고 회고는 사람·타임라인으로 나눠 담는다. 구성·라벨 정의는 [공통 기준](./README.md#12-공통-기준-전-화면-공유)에 있다.
- 그래프뷰가 탐색에 실제로 유용한지, 리스트뷰 대비 사용 빈도 검증 필요.
- 1년 전 오늘 회고가 반가움 vs 부담 중 어느 쪽으로 느껴지는지 검증 필요.

## 10. 사람 탭 (인물 디렉토리)

홈의 관계도가 *관계를 한눈에 보는* 곳이라면, 사람 탭은 *특정 사람을 찾고 관리하는* 인물 디렉토리다.

- **홈 리스트뷰와의 차이:** 홈 리스트뷰는 관계도의 목록 표현(태그 필터 공유, 개요용)이고, 사람 탭은 **검색·정렬·즐겨찾기 상단·사람 추가**에 초점을 둔다.
- **정렬:** `이름순(가나다, 기본)`과 `최근 만남순` 중 선택하고, `즐겨찾기 상단`(기본 켬)으로 둔다. 이름순은 한글 → 영문 → 숫자 순이며 대소문자를 구분하지 않는다.
- **검색:** 이름과 관계 유형으로 부분 일치 검색한다(대소문자·공백 무시). 초성 검색은 `[post-MVP]`.
- **사람 카드:** 사진(또는 이름 첫 글자)·이름·★(즐겨찾기)·관계 유형·마지막 만남 경과를 보인다. 누르면 사람 프로필로 간다. → [02](./02-person.md)
- **사람 추가:** 목록에서 바로 추가한다. → [02 사람 등록](./02-person.md#2-사람-추가--프로필-등록)
- **빈 상태:**
  - 사람이 없을 때: `"아직 기록한 사람이 없어요. 첫 사람을 추가해 관계를 남겨보세요."` + `[＋ 사람 추가]` ([공통 기준](./README.md#12-공통-기준-전-화면-공유))
  - 검색 결과가 없을 때: `"'{검색어}'에 해당하는 사람이 없어요."` + `[검색 지우기]`

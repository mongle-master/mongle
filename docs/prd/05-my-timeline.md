# 05. 전체 타임라인 (나의 통합 연대기)

> **SSOT:** 전체 타임라인(모든 사람과의 기록을 하나의 시간축으로 보는 전역 화면 = 하단 내비 `타임라인` 탭)의 단일 기준.
> 상위 배경·목표: [개요](./README.md) · 관련 화면: [03 사람별 타임라인](./03-timeline.md), [02 사람 프로필](./02-person.md), [04 기록 작성](./04-record.md), [01 홈](./01-home-dashboard.md)
> 공통 규칙(내비·칩·날짜·글자수·문구·이미지·접근성): [공통 기준](./README.md#12-공통-기준-전-화면-공유)

## 1. 목적

모든 사람과 함께한 기록을 하나의 시간축(최신 → 과거)으로 모아 회고한다. [03 사람별 타임라인](./03-timeline.md)이 *한 사람*과의 이야기라면, 이 화면은 *나의 전체* 흐름이다.

## 2. 화면 요소

- 하단 내비 `타임라인` 탭(전역 top-level 화면). 상단: 화면 타이틀 `타임라인` + 필터 진입.
- 타임블록 리스트 — 카드 표기 규칙은 [03](./03-timeline.md#4-입력필드-상세-스펙)과 동일(제목·카테고리·언제·왜/무엇을·사건 성격 아이콘).
  - **각 카드에 연결된 사람 표시** — 아바타 + 이름. 여러 명이면 `{대표 이름} 외 N명`. (전역 화면 전용)
- **월 단위 그룹 헤더**로 시간대를 구분(각 카드에는 일(日)을 표기).
- 기록 작성 진입 = 중앙 `＋`(내비). → [04](./04-record.md). (내비 구성은 [공통 기준](./README.md#12-공통-기준-전-화면-공유))
- 활동 흐름 차트는 두지 않는다(전 인물 집계 시각화는 후순위, [§7](#7-열린-질문--리스크)).

## 3. 상호작용

- 카드 본문을 누르면 기록 상세로 이동한다. → [04](./04-record.md)
- 카드의 **사람(아바타·이름)**을 누르면 그 인물 프로필로 이동한다. → [02](./02-person.md)
- 카테고리 태그를 누르면 그 카테고리로 필터한다(여러 개 = 합집합 OR).
- 상단 필터에서 **카테고리**와 **사람**으로 좁혀 본다.
- 중앙 `＋`로 기록을 작성한다. 전역 화면이라 연결할 사람이 정해져 있지 않으므로 **사람 선택부터** 시작한다(사람이 한 명도 없으면 사람 추가로 유도 — [공통 기준](./README.md#12-공통-기준-전-화면-공유)). → [04](./04-record.md)

## 4. 입력·필드 상세 스펙

> 제품 공통 확정 스펙. 카드 표기·글자수·값 집합·날짜 규칙은 [03](./03-timeline.md)·[공통 기준](./README.md#12-공통-기준-전-화면-공유)을 따른다(다시 정의하지 않는다).

| 항목 | 값/타입 | 제약·규칙 | 기본값 | 비고 |
|---|---|---|---|---|
| 카드 표기 | — | [03 타임블록 카드](./03-timeline.md#4-입력필드-상세-스펙) 규칙 재사용 | — | 다시 정의하지 않음 |
| 연결된 사람 표시 | 아바타 + 이름 | 1명: 아바타 + 이름 / 여러 명: `{대표 이름} 외 N명`. 대표 = 즐겨찾기 → 가나다 우선 1인 | — | 전역 전용. 다중 연결 규칙은 [04](./04-record.md) |
| 정렬 | 시간순 | **최신 → 과거** (같은 날은 나중에 있었던 일이 위로) | 최신순 | 03과 동일 |
| 그룹 | 월 단위 | 월 헤더 `YYYY년 M월`로 묶고, 카드에는 일(日)만 표기 | — | 03(사람별)은 헤더 없이 카드마다 날짜 표기 |
| 카테고리 필터 | 칩(다중) | 여러 개 = **합집합(OR)** | 전체(미선택) | 값 집합 [공통 기준](./README.md#12-공통-기준-전-화면-공유) |
| 사람 필터 | 인물 선택 | 특정 인물만. 여러 명 = 합집합(OR). 카테고리 필터와는 **교집합(AND)** | 전체(미선택) | 필터는 목록에만 적용 |

## 5. 상태·엣지케이스

- **기록 0건(전무)**: "아직 함께한 기록이 없어요. 첫 순간을 새겨보세요." + `[기록 작성]`
  - 단, 아직 사람이 **한 명도 없으면** 기록을 만들 수 없으므로 사람 추가로 유도한다: "먼저 함께한 사람을 추가해 주세요." + `[＋ 사람 추가]` (공통 기준)
- **필터 결과 0건**: "이 조건에 맞는 기록이 없어요." + `[필터 초기화]`
- **피드를 불러오지 못함**: "불러오지 못했어요. 잠시 후 다시 시도해 주세요." + `[다시 시도]`

## 6. 데이터 · 백엔드 연동

- 전체 타임라인 피드: 모든 인물의 기록을 하나의 시간축으로 시간순 집계. 기록 원천은 [04 기록 작성](./04-record.md)에서 생성/저장.
- 사람별 타임라인·활동 흐름은 [03](./03-timeline.md) — [#6](https://github.com/mongle-master/mongle/issues/6).

## 7. 열린 질문 / 리스크

- **회고(1년 전 오늘) 상시 노출 여부** — 정의·노출은 [01 홈](./01-home-dashboard.md)의 ‘1년 전 오늘’이 담당한다. 전체 타임라인은 순수 피드로 유지하며, 상단에 회고 카드를 상시 노출하는 것은 후순위(홈과 중복 노출 방지).
- **전역 활동 흐름 시각화**(전 인물 집계)는 후순위.

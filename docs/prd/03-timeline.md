# 03. 사람별 타임라인

> **SSOT:** 사람별 타임라인(이벤트 타임블록, 활동 흐름 시각화, 카테고리 필터)의 단일 기준.
> 상위 배경·목표: [개요](./README.md) · 관련 화면: [02 사람 프로필](./02-person.md), [04 기록 작성](./04-record.md), [05 전체 타임라인](./05-my-timeline.md)
> 공통 규칙(내비·칩·날짜·글자수·문구·이미지·접근성): [공통 기준](./README.md#12-공통-기준-전-화면-공유)
> 백엔드: [#6 사람별 타임라인 & 활동 흐름 API](https://github.com/mongle-master/mongle/issues/6)

## 1. 목적

특정 사람과 함께한 기록을 시간순으로 보고, 활동 흐름을 회고한다.

## 2. 화면 요소

- 상단 탭: `타임라인` / `프로필` (프로필 탭 → [02](./02-person.md#3-사람-프로필-조회))
- 통계 영역 — 함께한 기록 수 `{N}개`, 마지막 만남(상대시간), 그리고 헤더에 `{처음 만난 해}년부터 · {만난 횟수}번 만남`. 계산은 [02 자동 항목](./02-person.md#2-사람-추가--프로필-등록)과 동일 원천이며, **만난 횟수 = ‘만남’ 카테고리 기록의 고유 날짜 수**.
- **활동 흐름 차트** — 일반 막대/선 차트 대신 관계 기록에 맞는 **다른 형태의 시각화**(활동 종류 레인 × 월, [§4](#4-입력필드-상세-스펙)). (mock-timeline)
- 타임블록 리스트, 각 블록:
  - 제목 (없으면 자동 제목 `{이름} · {카테고리}` → [04](./04-record.md))
  - 카테고리 태그
  - 언제 (날짜, 시간 있으면 함께)
  - 왜 만났는지 / 무엇을 했는지
  - 사건 성격 아이콘 (카테고리에 따라)
- 상황 기록 작성 진입 → [04](./04-record.md) (이 인물이 연결 대상). 전역 하단 내비 구성은 [공통 기준](./README.md#12-공통-기준-전-화면-공유).

## 3. 상호작용

- 타임블록을 누르면 기록 상세로 이동한다(기존 값으로 열어 보기·수정). → [04](./04-record.md)
- 카테고리 태그를 누르면 해당 유형의 기록만 필터링한다. 여러 카테고리를 함께 고르면 **합집합(OR)**으로 넓혀 본다. (값 집합은 [04](./04-record.md#4-입력필드-상세-스펙)·[공통 기준](./README.md#12-공통-기준-전-화면-공유)과 공유)
- `상황 기록 작성`을 누르면 기록 작성 화면으로 이동한다. → [04](./04-record.md)

## 4. 입력·필드 상세 스펙

> 제품 공통 확정 스펙. 글자수·값 집합·날짜 규칙은 [공통 기준](./README.md#12-공통-기준-전-화면-공유)을 따른다(여기서 다시 정의하지 않는다).

| 항목 | 값/타입 | 제약·규칙 | 기본값 | 비고 |
|---|---|---|---|---|
| 정렬 | 시간순 | **최신 → 과거** (같은 날은 나중에 있었던 일이 위로) | 최신순 | 방향 확정 |
| 제목 | 텍스트 | 선택. 비우면 자동 제목 `{이름} · {카테고리}`로 표시. 리스트에서는 한 줄로 보여주고 길면 말줄임(…) | 자동 제목 | 원문 규칙 [04](./04-record.md) |
| 카테고리 태그 | 칩(단일) | 값 집합은 [공통 기준](./README.md#12-공통-기준-전-화면-공유)의 카테고리. 태그를 누르면 필터 | 만남 | 필수 |
| 언제 | 날짜(+선택 시간) | 표시 `YYYY.MM.DD (요일)`, 시간이 있으면 `HH:mm`도 함께. 기본 오늘, 미래 불가 | 오늘 | 날짜 규칙 [공통 기준](./README.md#12-공통-기준-전-화면-공유) |
| 왜 만났는지 / 무엇을 했는지 | 텍스트 | 리스트에서는 짧게(몇 줄) 보여주고 길면 말줄임(…). 값이 없으면 그 줄을 숨김 | — | 원문 각 100자 [04](./04-record.md) |
| 카테고리 필터 | 칩(다중) | 여러 개 동시 선택 = **합집합(OR)**, 하나라도 맞으면 노출 | 전체(미선택) | 필터는 목록에만 적용(차트·통계는 전체) |
| 활동 흐름 차트 | 관계 기록형 시각화 | 아래 ‘활동 흐름 차트(개념)’ | — | 우선순위 낮음 |
| 기록 0건일 때 | 빈 상태 | 안내 문구 + 기록 작성 유도 | — | [§5](#5-상태엣지케이스) |

**타임블록 카드 (표시 규칙)**
- 제목 · 카테고리 태그 · 언제 · 왜/무엇을 · 사건 성격 아이콘으로 구성. 왜/무엇을은 값이 있을 때만 노출한다.
- 선택적 강조 메모(mock의 ‘🔒 축의 10만원’ 같은 비공개 한 줄 강조)는 **초기 버전 이후 보강** — 기록 작성 폼에 해당 입력이 생기면 카드에 표시한다. 도입 시 짧은 한 줄(최대 20자).

**활동 흐름 차트 (개념)** — mock-timeline 근거, 우선순위 낮음(초기 이후 보강)
- **레인 3종**: `만남` · `연락` · `추억`. 각각 카테고리 `만남` · `연락` · `기념일` 기록에 대응한다.
- **가로축**: 월. 기본 최근 6개월.
- **표기**: 그 달에 그 레인 활동이 **있으면 점, 없으면 빈자리**로 둔다. 횟수·강도는 나타내지 않는다 — 관계를 점수화·서열화하지 않기 위함.
- **상호작용**: 점을 누르면 그 달 기록만 보도록 아래 목록을 좁힌다.
- **데이터가 적을 때(1~2건)**: 있는 달에만 점, 나머지는 빈자리. 기록이 **전혀 없으면 차트를 감춘다**. 최근 6개월에만 기록이 없으면 그 구간을 빈 상태로 두고 "최근 6개월은 조용했어요" 안내.
- `기타`·사용자가 추가한 카테고리는 기본적으로 차트에 넣지 않고 목록에만 노출한다([§7](#7-열린-질문--리스크)).

## 5. 상태·엣지케이스

- **기록 0건**: "아직 함께한 기록이 없어요. 첫 순간을 새겨보세요." + `[기록 작성]` (공통 기준 문구)
- **특정 카테고리 필터 결과 0건**: "이 조건에 맞는 기록이 없어요." + `[필터 초기화]`
- **활동 흐름 차트 데이터가 적을 때(1~2건)**: 있는 달에만 점, 나머지는 빈자리. 기록 0건이면 차트를 숨긴다([§4](#4-입력필드-상세-스펙)).
- **마지막 만남 표시 불가**(만남 기록/날짜 없음): 통계의 마지막 만남 값은 "기록 없음".

## 6. 데이터 · 백엔드 연동

- 사람별 타임라인 피드(요약 + 기록 시간순)와 활동 흐름 집계 — 백엔드 [#6](https://github.com/mongle-master/mongle/issues/6)
- 기록 원천: [04 기록 작성](./04-record.md)에서 생성·저장

## 7. 열린 질문 / 리스크

- **활동 흐름 시각화 형태** → **확정**: 활동 종류(만남/연락/추억) 레인 × 월에 유무를 점으로 표기하는 방식으로 결정. (근거: mock-timeline) 우선순위가 낮아 초기 버전 이후 보강한다.
- **타임블록 정렬 방향** → **확정**: 최신순(최신 → 과거).
- (남은 미결) `기타`·사용자가 추가한 카테고리를 활동 흐름 차트의 어느 레인에 넣을지 — 기본은 차트에서 제외(목록에만 노출). 시드 외 카테고리 비중이 커지면 재검토한다.

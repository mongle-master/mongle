# 02. 사람 (등록 · 프로필)

> **SSOT:** 인물 등록/수정, 사람 프로필(관계 요약·취향·태그·통계), 즐겨찾기의 단일 기준.
> 상위 배경·목표: [개요](./README.md) · 관련 화면: [01 홈](./01-home-dashboard.md), [03 타임라인](./03-timeline.md), [04 기록 작성](./04-record.md)
> 관련 기능: **F-03, F-04, F-07, F-09, F-12**
> 백엔드: [#3 인물 도메인 API](https://github.com/mongle-master/mongle/issues/3)

이 문서는 **등록/수정 화면(7.2)** 과 **조회 화면(7.3)** 을 함께 다룬다. 두 화면은 같은 인물 데이터를 공유하므로 필드 정의를 여기 한곳에 둔다.

---

## 2. 사람 추가 / 프로필 등록

**목적:** 새 인물을 등록하고, 이후 기록과 회고의 기준이 되는 기본 정보를 저장한다. (F-03)

**입력 항목**

- 프로필 사진
- 이름
- 생일
- 처음 만난 날
- 마지막 만난 날짜
- 관계 태그 chip N개 (F-07)
- 카테고리 태그 chip
- 어떤 관계인지 표시: 예) 회사 동료, 친구, 스터디원
- 좋아하는 것
- 싫어하는 것
- 즐겨찾기 여부 (F-09)

**자동 계산/표시 항목** (입력 아님 — 저장된 데이터로 파생)

- 처음 만난 날 기준 D+day
- 마지막 만난 날짜 기준 경과일
- 알고 지낸 시간
- 만난 횟수
- 새긴 기록 개수 → [04 기록](./04-record.md)에서 생성
- 최근 함께한 일 → [03 타임라인](./03-timeline.md)의 최신 이벤트

## 3. 사람 프로필 (조회)

**목적:** 특정 사람과의 관계 정보를 요약하고, 타임라인 또는 프로필 정보로 전환해 볼 수 있게 한다. (F-04)

**주요 요소**

- 상단 탭: `타임라인` / `프로필` (타임라인 탭 → [03](./03-timeline.md))
- 프로필 사진 / 이름 / 생일
- 처음 만난 날 / 마지막 만난 날짜 / D+day 표시
- 관계 chip 태그 N개 + 태그 기반 그룹화 표시
- 어떤 관계인지 표시: 예) 회사 동료
- 알고 지낸 시간 / 만난 횟수 / 새긴 기록 개수
- 최근 함께한 일
- 좋아하는 것 / 싫어하는 것
- 즐겨찾기 상태 (F-09)
- 프로필 수정 버튼 (F-12)

**상호작용**

- `타임라인` 탭 → 해당 사람의 기록 목록으로 전환. → [03](./03-timeline.md)
- `프로필` 탭 → 기본 정보와 관계 요약 표시.
- `프로필 수정` → 등록 정보 수정(위 [§2](#2-사람-추가--프로필-등록) 입력 항목 재사용).
- `상황 기록 작성` 진입 → [04](./04-record.md) (연결할 사람 = 이 인물 프리셋).
- 즐겨찾기를 켜면 홈 관계도 맵에서 테두리 강조. → [01](./01-home-dashboard.md#5-정책)

## 5. 즐겨찾기

- 즐겨찾기는 인물 단위 on/off 토글이다. (F-09)
- **효과(SSOT):** 켜면 홈 관계도 맵에서 해당 인물 **프로필 테두리**가 강조된다. 렌더링 규칙은 [01](./01-home-dashboard.md#5-정책).
- 리스트뷰에서의 즐겨찾기 표현: `TBD (협의)`

## 6. 정책

- **D+day:** 처음 만난 날 기준으로 표시한다.
- **마지막 만남:** 마지막 만난 날짜가 있으면 `마지막 만남 D+n` 또는 `n일 전`으로 표시한다.
- **태그:** 여러 개 선택 가능해야 한다.
- 관계를 점수화·서열화하는 표현은 사용하지 않는다.

## 9. 관련 사용자 흐름

**사람을 등록한다**
1. 홈에서 `+ 사람 추가`를 누른다. → [01](./01-home-dashboard.md)
2. 프로필 사진, 이름, 생일, 처음 만난 날, 관계 태그를 입력한다.
3. 좋아하는 것/싫어하는 것과 즐겨찾기 여부를 선택한다.
4. 저장 후 사람 프로필로 이동한다.

## 10. 열린 질문 / 리스크

- 관계 정보 직접 입력을 사용자가 부담스러워하지 않을지 검증 필요.
- 좋아하는 것/싫어하는 것이 "따뜻한 기록" vs "관리 대상"으로 느껴지는지 검증 필요.
- 좋아하는 것/싫어하는 것의 입력 형태(단일 텍스트 vs 다건 리스트) 확정 필요.

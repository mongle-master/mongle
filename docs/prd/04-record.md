# 04. 상황 기록 작성

> **SSOT:** 상황 기록 작성(제목·왜·무엇을·언제 + 감정·날씨·카테고리 칩 + 연결할 사람 + 사진, 저장→타임라인 반영)의 단일 기준.
> 상위 배경·목표: [개요](./README.md) · 관련 화면: [02 사람 프로필](./02-person.md), [03 타임라인](./03-timeline.md) · 공통 규칙: [공통 기준](./README.md#12-공통-기준-전-화면-공유)
> 백엔드: [#4 기록(이벤트) 도메인 API](https://github.com/mongle-master/mongle/issues/4)

## 1. 목적

사용자가 사람과 함께한 상황을 짧고 빠르게 기록한다. **30초 안에 완료**를 목표로 한다. 감정 하나만 골라도, 혹은 아무것도 적지 않아도 남길 수 있을 만큼 부담을 낮춘다.

## 2. TO-BE 입력 구조

하나의 통합 폼에 아래 항목을 담는다. 상세 제약은 [§4](#4-입력필드-상세-스펙).

- **텍스트:** 제목(한 줄) · 왜 · 무엇을(여러 줄) · 언제(날짜, 시간은 선택)
- **칩(개인화):** 감정(여러 개) · 날씨(하나) · 카테고리(하나, 필수·기본 `만남`)
- **연결할 사람:** 인물 선택(최소 1명, 진입 경로에 따라 프리셋) → [02](./02-person.md) · [03](./03-timeline.md)
- **사진:** 최대 5장

> 관계태그 칩은 인물(02)에 귀속하는 속성이라 이 화면에는 나타나지 않는다.

## 3. 화면 요소 · 상호작용

- **헤더 개인화 인사:** 연결된 사람이 한 명이면 `오늘 {이름}랑 어땠어요?`와 그 사람의 아바타, 그리고 `감정만 골라도 돼요. 세 줄이면 충분해요.` 안내를 보여준다. 여러 명이면 `오늘 어땠어요?`, 아직 아무도 연결되지 않았으면 `오늘 누구와 함께였어요?`로 사람 선택을 먼저 유도한다. (mock-record)
- 감정·날씨·카테고리 칩을 눌러 고른다(감정은 여러 개, 날씨·카테고리는 각각 하나). 카테고리는 제목 옆에 함께 보인다.
- 각 칩은 개인화 대상이다 — 새 칩을 추가하고, 이름을 바꾸면 지난 기록에도 반영되며, 지워도 이미 남긴 기록엔 남는다. 규칙은 [공통 기준](./README.md#12-공통-기준-전-화면-공유) 참조.
- 연결된 사람을 표시하고 추가·변경할 수 있다. 진입 경로에 사람이 있으면 미리 연결해 둔다.
- 사진을 첨부한다(최대 5장).
- 저장하면 연결된 **각 사람**의 [타임라인](./03-timeline.md)에 타임블록으로 즉시 추가된다.

## 4. 입력·필드 상세 스펙

| 항목 | 타입 | 필수 | 제약·검증 (글자수 등) | 기본값 | 비고 |
|---|---|---|---|---|---|
| 제목 | 한 줄 텍스트 | 선택 | 40자 이내 | 자동 제목 | 리스트·타임라인 카드 제목. 미입력 시 자동 제목 = `{연결한 사람} · {카테고리}` (여러 명이면 `{대표 이름} 외 N명 · {카테고리}`) |
| 왜 | 여러 줄 텍스트 | 선택 | 100자 이내 | — | 카드에 `왜 만났는지`로 표시 (mock-timeline). 무엇을과 독립 — 하나만 적어도 된다 |
| 무엇을 | 여러 줄 텍스트 | 선택 | 100자 이내 | — | 카드에 `무엇을 했는지`로 표시 (mock-timeline) |
| 언제 — 날짜 | 날짜 | 필수(기본 자동) | 오늘까지(미래 날짜 불가), 지난 날짜도 가능 | 오늘 | 타임라인 정렬·`1년 전 오늘` 회고(01)의 기준 |
| 언제 — 시간 | 시간 | 선택 | — | 없음 | 넣은 경우에만 카드에 시각을 함께 표시 |
| 감정 | 칩(다중 선택) | 선택 | 기록당 최대 5개. 값은 [공통 기준](./README.md#12-공통-기준-전-화면-공유) 참조 | 없음 | `감정만 골라도 되는` 가벼운 기록의 핵심 (mock-record) |
| 날씨 | 칩(단일 선택) | 선택 | 값은 [공통 기준](./README.md#12-공통-기준-전-화면-공유) 참조 | 없음 | |
| 카테고리 | 칩(단일 선택) | **필수** | 항상 1개 선택. 값은 [공통 기준](./README.md#12-공통-기준-전-화면-공유) 참조(`만남·연락·기념일·기타` + 사용자 추가) | 만남 | 제목 옆에 표시. 타임라인 카테고리 필터와 값 공유 → [03](./03-timeline.md) |
| 연결할 사람 | 인물 선택(다중) | **필수(최소 1명)** | 1명 이상 | 진입 경로의 인물 프리셋 → [02](./02-person.md) · [03](./03-timeline.md) | 여러 명 연결 시 각 사람의 타임라인에 모두 반영 |
| 사진 | 이미지(다중) | 선택 | 최대 5장. 세부 제약은 [공통 기준](./README.md#12-공통-기준-전-화면-공유) 참조 | 없음 | 첨부 순서 유지, 개별 삭제 가능 (mock-record) |

**최소 저장 조건:** 연결할 사람이 1명 이상이면 저장할 수 있다. 카테고리(기본 `만남`)와 날짜(기본 오늘)는 자동으로 채워지므로, 사람 프로필·타임라인에서 들어오면 화면을 열자마자 저장 가능하다. 제목·왜·무엇을·감정·날씨·사진은 모두 선택이다.

## 5. 입력 원칙

- 긴 일기보다 짧은 상황 기록을 우선한다.
- 사용자가 `왜 이 기록을 남기는지`, `무엇을 기억할지`만 빠르게 적게 한다. 둘 다 선택이며, 감정만 고르고 저장해도 된다.
- 기록 저장 후 연결된 사람의 타임라인에 즉시 반영한다. → [03](./03-timeline.md)

## 6. 상태·엣지케이스

| 상황 | 동작 · 문구 |
|---|---|
| 연결할 사람 0명 | 저장 전에 사람을 고르도록 안내한다: `함께한 사람을 한 명 이상 선택해 주세요.` |
| 감정만 골랐거나 아무것도 안 적음 | 연결할 사람이 1명 이상이면 저장할 수 있다(제목은 자동 생성). |
| 글자수 초과(제목 40자, 왜·무엇을 각 100자) | 한도를 넘으면 안내한다: `최대 N자까지 쓸 수 있어요.` (공통 문구 → [공통 기준](./README.md#12-공통-기준-전-화면-공유)) |
| 감정을 5개 넘게 고름 | 안내한다: `감정은 최대 5개까지 고를 수 있어요.` |
| 사진을 5장 넘게 첨부 | 안내한다: `사진은 최대 5장까지 넣을 수 있어요.` |
| 미래 날짜를 고름 | 안내한다(공통 문구 → [공통 기준](./README.md#12-공통-기준-전-화면-공유)). |
| 저장 실패 | 입력한 내용을 잃지 않게 유지하고 다시 시도하도록 안내한다(공통 문구 → [공통 기준](./README.md#12-공통-기준-전-화면-공유)). |
| 새 칩을 같은 이름으로 추가 | 중복은 만들지 않고 안내한다(공통 문구 → [공통 기준](./README.md#12-공통-기준-전-화면-공유)). |
| 작성 중 이탈(입력이 있는 경우) | 나가기 전에 확인한다: `작성 중인 내용이 있어요. 저장하지 않고 나갈까요?` (초안 임시저장은 `[post-MVP]`) |
| 등록된 사람이 없는데 ＋로 진입 | 사람부터 추가하도록 안내하고 [사람 추가](./02-person.md)로 이끈다(공통 빈상태 → [공통 기준](./README.md#12-공통-기준-전-화면-공유)). |

## 7. 데이터 · 백엔드 연동

- 저장하면 기록(이벤트)이 새로 생성된다([§4](#4-입력필드-상세-스펙)의 입력이 원천). — [#4](https://github.com/mongle-master/mongle/issues/4)
- 저장 후 연결된 **각 인물**의 파생값이 갱신되고 타임라인에 즉시 반영된다. — [#4](https://github.com/mongle-master/mongle/issues/4) · [03](./03-timeline.md)
  - **마지막 만남:** 이 기록의 날짜가 더 최근이면 갱신.
  - **함께한 기록 수:** 그 사람과 연결된 전체 기록 개수.
  - **만난 횟수:** `만남` 카테고리 기록의 **고유 날짜 수**(같은 날 여러 건은 1회로 센다).

## 8. 관련 사용자 흐름

**사람별 기록을 남긴다**

1. 사람 프로필 또는 타임라인에서 `상황 기록 작성`을 누른다(연결할 사람이 그 사람으로 미리 채워진다). 또는 하단 ＋에서 시작해 함께한 사람을 먼저 고른다. → [02](./02-person.md) · [03](./03-timeline.md)
2. 감정을 고르거나 제목·왜·무엇을를 짧게 적는다(모두 선택). 필요하면 날씨·사진·날짜/시간을 더한다.
3. 카테고리를 고른다(기본 `만남`).
4. 저장하면 연결된 사람(들)의 타임라인에 타임블록으로 추가되고, 관계 요약 통계가 갱신된다. → [03](./03-timeline.md) · [02](./02-person.md)

## 9. 열린 질문 / 리스크

- **왜/무엇을 2필드 구조:** 둘 다 선택 입력이며 서로 독립이다(하나만 적어도 됨). 부담을 줄이려 필수화하지 않는다. (남은 검증: 실제 사용에서 두 필드가 자연스러운지 관찰)
- **글자수 균형:** 제목 40자, 왜·무엇을 각 100자로 확정 — 짧은 상황 기록 목표에 맞춘 값.
- **카테고리 값 집합:** 인물·기록·타임라인 필터가 **하나의 카테고리 칩 세트를 공유**한다(별도 세트를 두지 않음). 정의는 [공통 기준](./README.md#12-공통-기준-전-화면-공유). → [02](./02-person.md) · [03](./03-timeline.md)
- **제목 필수/선택:** 선택 + 자동 제목으로 확정. 근거: `감정만 골라도 되는` 30초 기록을 막지 않기 위해서다. (mock-record)

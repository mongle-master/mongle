# 04. 상황 기록 작성

> **SSOT:** 상황 기록 작성(한 사람 기반 funnel + 감정·한 줄 기록 + 선택 상세 입력, 저장→타임라인 반영)의 단일 기준.
> 상위 배경·목표: [개요](./README.md) · 관련 화면: [02 사람 프로필](./02-person.md), [03 타임라인](./03-timeline.md) · 공통 규칙: [공통 기준](./README.md#12-공통-기준-전-화면-공유)
> 백엔드: [기록(event) 이슈](https://github.com/mongle-master/mongle/labels/event)

## 1. 목적

사용자가 사람과 함께한 상황을 짧고 빠르게 기록한다. **30초 안에 완료**를 목표로 한다. 감정 하나만 골라도, 혹은 아무것도 적지 않아도 남길 수 있을 만큼 부담을 낮춘다.

## 2. TO-BE 입력 구조

하나의 긴 폼이 아니라 `use-funnel` 기반 단계 입력으로 구성한다. 저장은 우상단 아이콘으로 고정하고, 다음 단계가 있을 때만 하단에 `이어서`를 둔다. 상세 제약은 [§4](#4-입력필드-상세-스펙).

- **1단계 — 사람:** 상단에 `{이름}님`만 표시하고, 큰 아바타로 기록 대상을 확인한다. 선택 UI는 두지 않는다.
- **2단계 — 마음:** `오늘은 ______ 다.` 문장과 inline 감정 칩을 보여준다.
- **3단계 — 장면:** 블랙앤화이트 톤의 편지지에 `[오늘 있었던 일]`만 둔다. 사진 추가도 이 단계에 둔다.
- **4단계 — 더 선명하게:** 종류와 언제만 필요할 때 더한다.

> 관계태그 칩은 인물(02)에 귀속하는 속성이라 이 화면에는 나타나지 않는다.

### 2.1. 복잡성을 만드는 요소와 개선 기준

- `더 적기` 같은 숨김 버튼으로 많은 입력을 접어두지 않는다. 숨겨야 할 만큼 많다면 funnel 단계로 나눈다.
- 괄호 안 설명으로 입력 의미를 보충하지 않는다. 화면 문장 자체가 다음 행동을 말해야 한다.
- 기록 화면에 일반 앱 header/footer를 두지 않는다. 목적이 분명한 풀스크린 작성 흐름으로 처리한다.
- `오늘의 감정` 같은 label은 먼저 의심한다. label보다 문장형 상태와 선택지 자체가 의미를 전달해야 한다.
- funnel progress indicator가 입력보다 먼저 눈에 들어오면 제거한다. 단계감보다 작성 목적이 먼저 보여야 한다.
- 함께한 사람은 한 명 기준으로 시작한다. 다중 선택 UI는 현재 범위에서 제거한다.
- 칩은 손가락으로 고르기 쉬운 크기로 두되, UI 전체가 과하게 커지지 않게 한다. 감정은 화면 폭을 채우는 fill이 아니라 inline chip으로 보여주고, chip 안의 문구에는 손글씨 폰트를 쓴다.
- 긴 textarea는 기본 박스처럼 보이지 않게 한다. 블랙앤화이트 톤의 편지지형 배경과 옅고 살짝 비뚤어진 가이드 라인을 쓴다.
- `저장/다음`을 좌우로 나란히 두지 않는다. 저장은 우상단 아이콘, 다음 단계 진입은 하단 `이어서` 하나로 둔다.
- 일반 UI 라벨·버튼·picker는 기본 UI 폰트를 쓴다. 손글씨 폰트는 대상 이름, 감정 chip 문구, 편지지형 입력처럼 기록 맥락을 만드는 텍스트에만 제한한다.
- 날짜와 시간은 네이티브 input 대신 카드형 date picker와 wheel형 time picker를 쓴다. 선택된 시각은 wheel 안의 선택 상태로만 보여주고 별도 큰 값으로 중복 표시하지 않는다.

## 3. 화면 요소 · 상호작용

- 기록 화면은 하단 탭 없이 열린다. 상단에는 닫기만 둔다.
- 상단에는 손글씨 폰트의 `{이름}님`과 우상단 저장 아이콘을 둔다.
- 사람 단계는 화면의 큰 아바타로 기록 대상을 확인한다. 아바타 아래 관계 정보는 한 개만 보여준다.
- 감정 칩은 `기뻤`, `감사했`, `편안했`처럼 `오늘은 ___다.` 문장에 붙는 형태로 보여주고, inline chip 안에 손글씨 문구를 직접 올린다.
- 감정·카테고리 칩을 눌러 고른다(감정은 여러 개, 카테고리는 하나). 칩은 한 손 조작 기준의 터치 영역을 확보하되, 글자 크기는 일반 UI 스케일을 유지한다.
- 각 칩은 개인화 대상이다 — 새 칩을 추가하고, 이름을 바꾸면 지난 기록에도 반영되며, 지워도 이미 남긴 기록엔 남는다. 규칙은 [공통 기준](./README.md#12-공통-기준-전-화면-공유) 참조.
- 한 줄 기록은 편지지형 배경의 입력으로 제공한다. 120자까지 쓰고, 첫 줄 일부가 자동 제목이 된다. 120자를 적을 수 있게 편지지 지면은 충분히 크게 둔다.
- 연결된 사람은 한 명이다. 진입 경로에 사람이 있으면 그 사람, 없으면 등록된 첫 번째 사람을 기본 기록 대상으로 둔다.
- 사진은 한 줄 기록 편지지 아래의 큰 `+` 영역에서 첨부한다(최대 5장).
- 저장하면 연결된 사람의 [타임라인](./03-timeline.md)에 타임블록으로 즉시 추가된다.

## 4. 입력·필드 상세 스펙

| 항목        | 타입            | 필수            | 제약·검증 (글자수 등)                                                                                              | 기본값                                                                                            | 비고                                                                          |
| ----------- | --------------- | --------------- | ------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| 제목        | 자동 생성       | 자동            | 첫 줄 기준, 줄바꿈 전까지만 사용                                                                                   | 한 줄 기록의 첫 줄 일부                                                                           | 별도 제목 input은 두지 않는다                                                 |
| 한 줄 기록  | 편지지형 텍스트 | 선택            | 120자 이내                                                                                                         | —                                                                                                 | 빠른 기록의 중심 입력. 저장 payload의 `memo`로 사용                           |
| 언제 — 날짜 | 날짜            | 필수(기본 자동) | 오늘까지(미래 날짜 불가), 지난 날짜도 가능                                                                         | 오늘                                                                                              | 타임라인 정렬·`1년 전 오늘` 회고(01)의 기준                                   |

| 감정        | 칩(다중 선택)   | 선택            | 기록당 최대 5개. 값은 [공통 기준](./README.md#12-공통-기준-전-화면-공유) 참조                                      | 없음                                                                                              | `감정만 골라도 되는` 가벼운 기록의 핵심 (mock-record)                         |
| 카테고리    | 칩(단일 선택)   | 자동            | 항상 1개 선택. 값은 [공통 기준](./README.md#12-공통-기준-전-화면-공유) 참조(`만남·연락·기념일·기타` + 사용자 추가) | 만남                                                                                              | 상세 단계에만 둔다. 타임라인 카테고리 필터와 값 공유 → [03](./03-timeline.md) |
| 연결할 사람 | 인물 1명        | **필수**        | 1명                                                                                                                | 진입 경로의 인물 프리셋, 없으면 첫 번째 등록 인물 → [02](./02-person.md) · [03](./03-timeline.md) | 다중 선택 UI는 현재 범위에서 제거                                             |
| 사진        | 이미지(다중)    | 선택            | 최대 5장. 세부 제약은 [공통 기준](./README.md#12-공통-기준-전-화면-공유) 참조                                      | 없음                                                                                              | 한 줄 기록 단계에서 첨부, 첨부 순서 유지, 개별 삭제 가능                      |

**최소 저장 조건:** 연결할 사람이 1명 있으면 저장할 수 있다. 카테고리(기본 `만남`)와 날짜(기본 오늘)는 자동으로 채워지므로, 사람 프로필·타임라인에서 들어오면 화면을 열자마자 저장 가능하다. 한 줄 기록·감정·사진은 모두 선택이다.

## 5. 입력 원칙

- 긴 일기보다 짧은 상황 기록을 우선한다.
- 사용자가 `무엇을 기억할지`만 빠르게 적게 한다. 감정만 고르고 저장해도 된다.
- 저장은 우상단 아이콘으로 둔다. 다음 단계가 있을 때만 하단에 `이어서`를 둔다.
- 기록 저장 후 연결된 사람의 타임라인에 즉시 반영한다. → [03](./03-timeline.md)

## 6. 상태·엣지케이스

| 상황                             | 동작 · 문구                                                                                                                |

| 저장 실패                        | 입력한 내용을 잃지 않게 유지하고 다시 시도하도록 안내한다(공통 문구 → [공통 기준](./README.md#12-공통-기준-전-화면-공유)). |
| 새 칩을 같은 이름으로 추가       | 중복은 만들지 않고 안내한다(공통 문구 → [공통 기준](./README.md#12-공통-기준-전-화면-공유)).                               |
| 작성 중 이탈(입력이 있는 경우)   | 나가기 전에 확인한다: `작성 중인 내용이 있어요. 저장하지 않고 나갈까요?` (초안 임시저장은 `[post-MVP]`)                    |

## 7. 데이터 · 백엔드 연동

- 저장하면 기록(이벤트)이 새로 생성된다([§4](#4-입력필드-상세-스펙)의 입력이 원천). — [event](https://github.com/mongle-master/mongle/labels/event)
- 저장 후 연결된 인물의 파생값이 갱신되고 타임라인에 즉시 반영된다. — [event](https://github.com/mongle-master/mongle/labels/event) · [03](./03-timeline.md)
  - **마지막 만남:** 이 기록의 날짜가 더 최근이면 갱신.
  - **함께한 기록 수:** 그 사람과 연결된 전체 기록 개수.
  - **만난 횟수:** `만남` 카테고리 기록의 **고유 날짜 수**(같은 날 여러 건은 1회로 센다).

## 8. 관련 사용자 흐름

**사람별 기록을 남긴다**

1. 사람 프로필 또는 타임라인에서 `상황 기록 작성`을 누른다(연결할 사람이 그 사람으로 미리 채워진다). 진입 경로에 사람이 없으면 등록된 첫 번째 사람을 대상으로 시작한다. → [02](./02-person.md) · [03](./03-timeline.md)
2. `{이름}님` 상단 제목과 큰 아바타로 기록 대상을 확인한다.
3. `오늘은 ______ 다.` 문장에 맞는 마음 칩을 고르거나 바로 다음으로 간다.
4. 편지지형 입력에 한 줄 기록을 남기고, 필요하면 사진을 더한다.
5. 필요하면 종류와 날짜/시간을 더한다.
6. 저장하면 연결된 사람의 타임라인에 타임블록으로 추가되고, 관계 요약 통계가 갱신된다. → [03](./03-timeline.md) · [02](./02-person.md)

## 9. 열린 질문 / 리스크

- **다중 사람 기록:** 현재 모바일 작성 흐름에서는 제거한다. 필요해지면 별도 확장으로 다룬다.
- **글자수 균형:** 한 줄 기록 120자로 확정 — 짧은 상황 기록 목표에 맞춘 값.
- **카테고리 값 집합:** 인물·기록·타임라인 필터가 **하나의 카테고리 칩 세트를 공유**한다(별도 세트를 두지 않음). 정의는 [공통 기준](./README.md#12-공통-기준-전-화면-공유). → [02](./02-person.md) · [03](./03-timeline.md)
- **제목 필수/선택:** 별도 입력은 제거하고 자동 제목으로 확정. 근거: 기록 입력 surface를 줄이기 위해서다.

# 04. 상황 기록 작성

> **SSOT:** 상황 기록 작성(제목/왜/무엇을 입력, 카테고리 chip, 연결할 사람, 저장→타임라인 반영)의 단일 기준.
> 상위 배경·목표: [개요](./README.md) · 관련 화면: [02 사람 프로필](./02-person.md), [03 타임라인](./03-timeline.md)
> 관련 기능: **F-06, F-07**
> 백엔드: [#4 기록(이벤트) 도메인 API](https://github.com/mongle-master/mongle/issues/4)

## 1. 목적

사용자가 사람과 함께한 상황을 짧고 빠르게 기록한다. **30초 안에 완료**를 목표로 한다.

## 2. TO-BE 입력 구조

- **제목:** input box 1개
- **왜:** text box 1개
- **무엇을:** text box 1개
- **카테고리 태그 chip** (F-07)
- **연결할 사람** (진입 경로에 따라 프리셋 가능 → [02](./02-person.md), [03](./03-timeline.md))

## 3. 화면 요소 · 상호작용

- 제목 우측에 카테고리 tag 노출
- chip 추가/선택
- 연결된 사람 표시
- 저장 버튼
- 저장 시 해당 사람의 [타임라인](./03-timeline.md)에 타임블록으로 즉시 추가된다.

## 4. 입력·필드 상세 스펙

> 🖊️ **이 표가 이 화면의 핵심 구체화 영역.** 글자수 제한·필수 여부·검증을 개발자가 채운다. 미정은 `TBD (협의)`.

| 항목 | 타입 | 필수 | 제약·검증 (글자수 등) | 기본값 | 비고 |
|---|---|---|---|---|---|
| 제목 | input 1줄 | **필수(제안)** | 최대 글자수 `TBD` | — | 리스트 노출 제목 |
| 왜 | text box | 선택 | 최대 글자수 `TBD` (예: 30자 등) | — | "왜 남기는지" |
| 무엇을 | text box | 선택 | 최대 글자수 `TBD` | — | "무엇을 기억할지" |
| 카테고리 태그 | chip | 선택 | 값 집합 `TBD` (만남/연락/기념일/기타 등), 단일/다중 `TBD` | — | [03 필터](./03-timeline.md#3-상호작용)와 값 공유 |
| 연결할 사람 | 인물 선택 | **필수** | 1명/다중 `TBD` | 진입 인물 프리셋 | [02](./02-person.md) |
| 날짜 | 날짜 | TBD | 기본 오늘 여부 `TBD` | 오늘(제안) | [03] 정렬 기준 |

> 예시로 남긴 "왜 30자 제한" 같은 값은 확정치가 아니라 **채워 넣을 자리**다. 확정되면 이 표를 갱신한다.

## 5. 입력 원칙

- 긴 일기보다 짧은 상황 기록을 우선한다.
- 사용자가 `왜 이 기록을 남기는지`, `무엇을 기억할지`만 빠르게 적게 한다.
- 기록 저장 후 해당 사람의 타임라인에 즉시 반영한다. → [03](./03-timeline.md)

## 8. 관련 사용자 흐름

**사람별 기록을 남긴다**
1. 사람 프로필 또는 타임라인에서 `상황 기록 작성`을 누른다. → [02](./02-person.md) · [03](./03-timeline.md)
2. 제목, 왜, 무엇을 입력한다.
3. 카테고리 tag chip을 선택한다.
4. 저장하면 사람별 타임라인에 타임블록으로 추가된다. → [03](./03-timeline.md)

## 9. 열린 질문 / 리스크

- `왜`/`무엇을` 2필드 구조가 기록 작성에 자연스러운지 검증 필요.
- 각 필드 글자수 제한이 "30초 기록" 목표와 균형이 맞는지 검증 필요.
- 카테고리 값 집합을 [02 인물 카테고리 태그](./02-person.md#4-입력필드-상세-스펙)와 통일할지, 별도로 둘지 결정 필요.

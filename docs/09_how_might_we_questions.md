# Step 09. "어떻게 하면 ~" 질문 만들기

> 목적: 서비스 방향을 맞춘 뒤, **더 구체적인 생각**으로 수렴한다.

> **Guide:**  
> 1. Agent가 `docs/04`, `docs/06`, `docs/08`을 읽고 **"어떻게 하면 ~ 할 수 있을까?"** 형태의 질문을 만든다. 직접 질문을 추가해도 좋다.
> 2. 팀이 질문을 보며 답을 이야기한다.
> 3. 각 질문 아래 **답**을 적는다. (타이머 ON)  
> - 질문은 **많을수록** 좋고, 하고 싶은 말이 많은 질문이 좋다.  
> - **실제로 어떻게 할지** 떠올릴 수 있는 질문을 한다. (예: 사용자를 어떻게 모을까? → 너무 넓음)  
> - 답은 **질문과 같은 말 반복·뭉뚱그린 표현 금지**. 구체적인 문장으로 쓴다.  
>   - (X) 어떻게 하면 사용자가 이 기능을 더 잘 쓸까? → 사용자들이 더 쉽게 쓰게 만들면 될 것 같아요  
> - 참고한 서비스·자료가 있으면 답에 적는다.

**프롬프트 예시**

```
docs/04_service_goal_target_value.md, docs/06_pain_wow_points.md, docs/08_word_cloud.md를 읽고
docs/09_how_might_we_questions.md의 "Agent가 만든 질문" 섹션만 채워줘.

질문 규칙:
- "어떻게 하면 ~ 할 수 있을까?" 형태
- docs/04 목적·핵심 가치, docs/06 불편한 점·좋았던 점, docs/08 키워드를 골고루 반영
- 실제 방법·아이디어가 떠오를 수 있는 질문 (너무 넓은 "사람 모으기"류 X)
- 최소 10개 이상, 서로 겹치지 않게
- 각 질문 아래 답 칸은 비워둬 (팀이 채움)
- "팀이 추가한 질문" 섹션은 건드리지 마
- 구조와 상단 Guide는 유지
```

## Agent가 만든 질문

> `docs/04`(목적·가치), `docs/06`(불편한 점·좋았던 점), `docs/08`(키워드)을 바탕으로. 아래 답은 Day1 워크숍에서 팀이 낸 아이디어.

### 질문 1: 어떻게 하면 사용자가 직접 많이 기록하지 않아도 관계가 쌓이게 할 수 있을까?

- 입력 자동화 — import from Slack MCP(오늘 해봄) / 카카오 대화내역 불러오기 (단, 여러 관계 일괄은 아님)
- GPS 기반으로 현재 위치를 자동 입력
- input 지면을 조금만 줘서 많이 안 쓰게 만들기

### 질문 2: 어떻게 하면 만남 직후 그때의 이야기·감정을 부담 없이 짧게 기록하게 할 수 있을까?

- 쉽게 쓰는 건 **감정 chip**을 주고, 조금 긴 건 **세 줄** 정도로 제한 (제한하면 압축해서 씀)
- 메인은 select/chip으로 30초 안에, 줄글 형식은 추가 기능으로

### 질문 3: 어떻게 하면 연락이 뜸해진 인물을 사용자가 스스로 알아차리게 할 수 있을까?

- 관계 지도를 주기적으로 열어보게 하고, 연락/만남 주기를 분석해 푸시
- "이 사람이랑 연락 안 한 지 오래됐어요" 알림 / 비활성 관계(긴 주기 친구) 점검

### 질문 4: 어떻게 하면 멀어진 사람에게 다시 연락할 계기를 만들 수 있을까?

- "1년 전 오늘", "작년 이맘때 이 사람과 뭐 했어요" 상기
- 생일뿐 아니라 기념일이 다가오는 사람 알림 → 스쳐 지나간 인연을 나중에 꺼내볼 계기

### 질문 5: 어떻게 하면 한 인물과의 관계에 사진·메모·만남을 자연스럽게 연결지을 수 있을까?

- 인물별 타임라인/사진첩 — 인물을 누르면 만난 일자·날짜를 목록·카드로 나열
- 사진 말고 **그림일기**로 기억하는 방법도

### 질문 6: 어떻게 하면 많은 관계를 쉽게(한눈에) 보여줄 수 있을까?

- 주요 인물 위주로 (가장 오래 지낸 관계 / 새로 추가된 관계 / 내가 중요하다고 느끼는 관계)
- 첫 페이지에서 인물별 대시보드처럼 / 관계 리포트(주간 회고)

### 질문 7: 어떻게 하면 관계를 시간의 흐름으로 보여줄 수 있을까?

- 사람별 타임라인 + 전체 인물 통합 타임라인(구간별) + 타임라인 맵
- 관계도 자체를 흐름(가까워짐/멀어짐)으로 시각화

### 질문 8: 어떻게 하면 화면을 구성해야 내가 어떤 성향인지 쉽게 이해할 수 있을까?

- 내가 쓴 내용을 바탕으로 나의 관계 성향을 **직관적으로** 표현 (자동 파악)

### 질문 9: 어떻게 하면 긴 시간에 걸친 내역을 쉽게 이해할 수 있을까?

- 일부 구간에 대해 모든 사람 타임라인 / 사람별 타임라인 / 감사일기 형태

### 질문 10: 어떻게 하면 그때의 만남을 "의미 있게" 기록할 수 있을까? (의미의 기준은?)

- 물질적·감정적 등 사람마다 기준이 달라서, "의미"의 보기(뷰)를 여러 개 만들어 두기

### 질문 11: 어떻게 하면 관계를 점수화·분석하지 않으면서도 인물별 기록을 자유롭게 남기게 할 수 있을까?

- 점수화 대신 정성(감정 분석) + 정량(상호작용) 을 은근하게, 압박 없는 도감 느낌으로

### 질문 12: 어떻게 하면 사용자가 따뜻함을 느낄 수 있을까? (앱 소개·온보딩·첫 메인)

- 첫인상에서 느낀 "따뜻함·인류애·힐링" 키워드를 뽑아 UI/UX를 따뜻하게 [체다]

### 질문 13: 어떻게 하면 많은 사람들이 공감하며 이용하게 할 수 있을까? (공감 포인트 형성)

- "그때 그 친구 어떻게 지낼까"의 보편 정서를 온보딩·소개에서 건드리기

### 질문 14: 어떻게 하면 생일뿐 아니라 관계의 맥락(최근 만남·추억)을 챙기게 할 수 있을까?

- 생일 알림에 최근 만난 시점·함께한 추억을 함께 띄워 맥락 있는 챙김으로

### 질문 15: 어떻게 하면 프라이빗과 공유의 기준을 나눌 수 있을까?

- 기본은 프라이빗 일기, 친한 사람만 초대해 공동 공간 공유 / 공개 범위 설정

### 질문 16: 어떻게 하면 그 사람과 있었던 일을 쉽게 볼 수 있을까?

- 우선 Timeline·사진첩으로, 인물을 누르면 카드 형태로

### 질문 17: 어떻게 하면 내 감정을 돌볼 수 있을까?

- 감사일기 / 감정 분석으로 나 자신과의 관계도 챙기기

### 질문 18: 어떻게 하면 바이럴 될 수 있을까? (핵심 기능으로 정체성 전달)

- 핵심 기능 하나로 정체성을 쉽고 간단히 전달 (프로토타입 후 논의) — "일기장 포지션"이 명확
- 명확한 목적성보다 기능적 접근이 더 명확할 수 있음 (예: 일기는 목적은 모호해도 기능은 명확)

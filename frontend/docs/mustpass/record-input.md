# mustpass — 기록 작성 입력 (record.tsx)

> 기준: [PRD 04-record](../../../docs/prd/04-record.md). 이 목록이 깨지면 회귀다.

## 저장 계약

- [ ] 함께한 사람 0명이면 저장 불가 + `함께한 사람을 한 명 이상 선택해 주세요.` 안내.
- [ ] 사람 1명 이상 + 날짜(기본 오늘)면 **아무것도 안 적어도 저장된다.** (감정만/무텍스트 저장 허용)
- [ ] 제목 미입력 시 자동 제목 `{대표 이름}{ 외 N명} · {카테고리}` 로 저장된다. (PRD §4)
- [ ] 카테고리 기본 `만남`, 날짜 기본 오늘이 자동으로 채워진 상태로 진입한다.

## 필드 제약 (PRD §4·§6)

- [ ] 제목 40자, 왜·무엇을 각 100자 초과 시 저장 차단 + 안내.
- [ ] 감정 최대 5개 초과 선택 불가 + `감정은 최대 5개까지 고를 수 있어요.`
- [ ] 사진 최대 5장 초과 불가 + 안내.
- [ ] 미래 날짜 선택 시 저장 차단 + 안내.

## PRD 필수 입력 노출 (드리프트 해소)

- [ ] 감정 칩(다중)·날씨 칩(단일)이 화면에 실제로 노출·저장된다. (기존 하드코딩 `emotionChipIds:[]`/`weatherChipId:null` 제거)
- [ ] 감정은 "감정만 골라도 저장"을 살리기 위해 1차 화면(스크롤 없이 닿는 위치)에 있다.

## 수정(edit) 모드

- [ ] `?eventId=` 진입 시 기존 값(사람·제목·왜·무엇을·날짜·시각·카테고리·감정·날씨·사진)이 폼에 복원된다.

## 회귀 방지

- [ ] 저장 성공 시 연결된 첫 사람의 타임라인으로 이동, 실패 시 로컬 처리 후 이동(기존 동작 유지).
- [ ] 사진은 여러 장 한 번에 첨부해도 최대 5장 상한을 넘지 않는다.

## 퍼널(use-funnel) 몰입형 흐름

- [ ] 기록 화면은 앱 헤더·하단 탭바 없이 전체화면으로 뜬다(몰입형). AppScreen 래핑 필수 — 없으면 push 되어도 탭 화면을 덮지 못해 + 버튼이 무반응으로 보인다. 전환은 인물 화면 진입(personId 프리셋)이면 push 슬라이드, 그 외(하단 ＋·기록 수정)는 fullScreen 모달.
- [ ] 단계: `감정` → `무엇을` → `세부`. 프리셋/수정 진입은 감정부터, ＋무프리셋 진입은 `사람`(전화번호부식) 단계부터.
- [ ] 모든 단계에서 `완료`(저장)가 상시 가능하다(사람만 있으면 저장 조건 충족). 마지막 단계는 `완료`만.
- [ ] 브라우저 뒤로가기로 이전 단계로 이동한다(TanStack Router와 충돌·에러 없음). 첫 단계에서 뒤로가기는 화면을 벗어난다.
- [ ] 단계 전환 시 URL의 `personId`/`eventId`가 보존된다.
- [ ] `완료`(저장)는 상단 우측에 상시. 하단은 `이어서`(다음)만, 마지막 단계는 하단 CTA 없음.
- [ ] 제목 인풋은 없다. 본문(무엇을) 첫 줄이 제목이 된다.
- [ ] 감정은 최대 5개 다중 선택, 표기는 "오늘은 ___다"에 맞는 과거형(`EMOTION_PAST` 매핑)으로 통일 — 매핑에 없는 라벨은 그대로 노출된다. `언제`는 날짜=가로 스트립(과거 30일), 시간=24시간 시/분 휠(오전오후 없음).
- [ ] 본문은 백엔드 `memo` 단일 필드로 저장(main에서 왜/무엇을 → 메모 통합). 무엇을 편지지 = memo, 왜 칩은 제거.
- [ ] 저장은 상단 우측 저장(Save) 아이콘.
- [ ] 손글씨 서체(온글잎 은별)는 유저 입력·감정 텍스트에만. UI(헤더·라벨·버튼·휠)는 기본 서체.

# mustpass: 더미 데이터 모드

프론트가 백엔드 없이도 화면을 보여줄 수 있는 명시적 모드. `FE_DUMMY_DATA=true`일 때만 켜진다.

## 불변식

1. **실 API와 더미 데이터는 절대 섞이지 않는다.**
   - 더미 모드 off(기본): 더미 데이터 코드 경로가 전혀 실행되지 않는다. API 실패는 에러로 표출한다(조용한 대체 금지).
   - 더미 모드 on: 네트워크 호출이 0건이다. 일부만 더미로 대체하는 상태는 존재하지 않는다.
2. **분기는 `lib/api/*` 진입점에서만 한다.** 라우트·컴포넌트는 모드의 존재를 모른다.
3. **더미 모드에서 쓰기는 저장되지 않고, 저장된 척하지 않는다.** 생성·수정·삭제·업로드는 `DummyModeError`를 던지고 폼이 안내 문구를 보여준다.
4. **쓰기 payload의 칩·인물 id는 반드시 서버 응답에서 온 값이어야 한다.** 하드코딩 id가 실 백엔드로 제출되는 순간 NOT_FOUND·데이터 유실이 난다(2026-07-11 사고 원인).
5. **저장 실패를 성공처럼 보이게 하지 않는다.** 실패 시 에러를 표시하고 입력 화면에 머문다(입력 내용 보존).
6. **더미 데이터는 백엔드 시더와 같은 시나리오를 유지한다.** `DemoDataSeeder`/`ChipSeeder`(인물 5명·기록 8건·칩 라벨)가 바뀌면 `lib/dummy-data.ts`도 같이 고친다. id 값은 네트워크를 타지 않으므로 일치시킬 필요 없다.
7. **더미 모드가 켜져 있음을 화면에서 알 수 있어야 한다**(뱃지). 실데이터로 오인 금지.

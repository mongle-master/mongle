# mustpass — 사람 등록 입력 (person-new-activity.tsx)

> 기준: [PRD 02-person](../../../docs/prd/02-person.md). 이 목록이 깨지면 회귀다.

## 저장 계약

- [ ] 이름 없으면 저장 불가 + 안내. **이름만 있으면 어느 단계에서든 저장된다.** (그 외 입력 전부 선택 — PRD §4)
- [ ] 처음 만난 날은 선택. 단 월·일만 넣고 연도를 비우면 저장 차단 + `처음 만난 날의 연도를 입력해 주세요.`
- [ ] 저장 성공 시 새 인물 프로필로 **replace** 이동(뒤로가기 시 등록 폼이 다시 나오지 않는다).
- [ ] 저장 실패 시 안내 후 같은 자리에서 재시도할 수 있다.

## 필드 제약 (PRD §4·§7)

- [ ] 이름 최대 20자. 동명이인 허용(중복 검사 없음).
- [ ] 관계 태그 최대 10개 초과 불가.
- [ ] 좋아하는 것·조심할 것 각 최대 20개, 항목 30자, 같은 항목 중복 금지.
- [ ] 생일은 월·일 동반 필수, 연도는 선택, 미래 불가. 처음 만난 날 미래 불가.
- [ ] 관계 유형은 현 구현 기준 최대 10자(PRD §4는 20자 — 미해결 drift, 이 재설계에서 건드리지 않음).

## 퍼널(use-funnel) 흐름

- [ ] 단계: `이름` → `관계` → `날짜` → `세부`.
- [ ] 이름 단계는 이름 입력 전까지 하단 `다음` 비활성, 상단 저장 아이콘 없음.
- [ ] 관계 단계부터 상단 우측 저장 아이콘 상시 노출. 마지막 단계는 하단 CTA 없이 저장 아이콘만.
- [ ] 브라우저 뒤로가기로 이전 단계 이동. 첫 단계에서 뒤로가기는 화면 이탈.
- [ ] 단계를 오가도 입력값이 유지된다(초기화 금지).
- [ ] 단계 전환에 방향성 슬라이드(reduced-motion 시 페이드). 헤더·하단 CTA는 고정, 콘텐츠만 밀린다.
- [ ] 처음 만난 날·생일은 `날짜 입력 / 입력 안 함` 토글 + 연/월/일 휠. 기본은 입력 안 함. 생일 연도는 `모름` 선택 가능(월·일만 저장).

## 수정(edit) 모드

- [ ] 수정 화면(person-edit-activity)은 기존 단일 폼(PersonForm) 유지 — 이 재설계는 등록만 대상.

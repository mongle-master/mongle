# mustpass — 스택 내비게이션 (src/stackflow/)

> 기준: PRD README §12.1(하단 내비), 각 화면 PRD의 진입/복귀 흐름. 이 목록이 깨지면 회귀다.

## 탭 (Main activity)

- [ ] `/` 진입 시 `/home`(홈 탭)으로 리다이렉트된다.
- [ ] 탭 전환(홈↔몽글라인↔사람↔설정)은 URL(`/home`·`/timeline`·`/people`·`/settings`)을
      바꾸되 **히스토리 엔트리를 만들지 않는다** (탭 여러 번 전환 후 뒤로가기 한 번 = 앱 이전 화면).
- [ ] 탭 전환 후 돌아와도 탭 내부 상태(검색어·필터·스크롤)가 남아 있다 (Main 미언마운트).
- [ ] 방문하지 않은 탭은 마운트되지 않는다 (첫 진입에 모든 탭 쿼리가 동시에 나가지 않는다).

## push 전환 (스펙: list item 선택·기록·폼)

- [ ] 사람 목록 아이템 / 홈 관계맵 노드 / 홈 회고 플로팅 / 타임라인 카드 / 이벤트 상세의
      인물 칩 → 각각 인물·이벤트 상세가 **push**(슬라이드 인)된다.
- [ ] 하단 ＋, 인물 프로필의 `상황 기록 작성`, 이벤트 상세의 `수정` → 기록 화면이
      **아래에서 위로**(iOS 모달 present) 뜨고, 아래 화면(탭 선택·상태)이 유지된다. (PRD 12.1)
- [ ] 인물 프로필의 `프로필 수정`, 사람 탭의 `추가` → 폼 화면 push.
- [ ] 브라우저 뒤로가기·iOS 엣지 스와이프백 = pop. 어느 화면에서 push했든 **한 번에**
      원래 화면·상태로 돌아온다 (returnTo 류 param 없음).

## step 전환 (화면 push 아님)

- [ ] 인물 프로필↔타임라인 토글은 URL 쿼리(`?view=timeline`)만 바뀌고 히스토리가 늘지 않는다.
      프로필로 돌아오면 쿼리가 사라진다.

## 저장·삭제 후 랜딩

- [ ] 새 기록 저장 → 첫 연결 사람의 타임라인으로 **replace** (뒤로가기 시 기록 폼이 다시 안 나옴).
- [ ] 기록 수정 저장 → 이벤트 상세로 pop, 수정 내용 반영.
- [ ] 사람 등록 → 새 인물 프로필로 replace. 프로필 수정 저장 → 프로필로 pop.
- [ ] 인물 삭제(수정 화면) → 프로필까지 두 단계 pop.

## URL·딥링크

- [ ] 기존 URL 전부 동작: `/`, `/timeline`, `/people`, `/settings`, `/record`,
      `/people/new`, `/people/:id`, `/people/:id/edit`, `/events/:id`,
      그리고 구 `/people/:id/timeline` → `/people/:id?view=timeline` 리다이렉트.
- [ ] 상세 URL 직접 진입(새로고침·공유) 시 화면이 뜨고, 뒤로가기가 앱 이탈이 아니라
      해당 탭 화면으로 떨어진다 (defaultHistory).
- [ ] 매칭 안 되는 다중 세그먼트 URL은 NotFound, 알 수 없는 단일 세그먼트는 홈 탭으로 흡수.

## 데스크톱

- [ ] PC 뷰포트에서도 화면·슬라이드 전환이 가운데 `max-w-md`(448px) 컨테이너 안에서만 일어난다.
